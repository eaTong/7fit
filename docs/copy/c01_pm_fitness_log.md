# C-1 · 一个产品经理的健身自救 · 文案稿 v3

> 视频类型：**C · 七练解码**
> 内容方向：**身份反差 + 造轮子（混合）**
> 目标平台：抖音 / 小红书
> 时长锚点：主体 ~70s / 全文 ~90s / 中速 3.4 字/秒
> BGM 选型：Cyber Pulse 增强版（synthwave, 100 BPM）
> 钩子类型：身份代入 + 反常识
> 北极星引用：「用产品思维去健身，用健身改造产品。」[来源: opc-niche.md §北极星]
> **v3 呈现**：**口播为主 + 信息密度拉满**——人在屏幕一侧讲，其余空间全部被信息卡片/数据/高亮填满。对标苹果发布会/科技博主信息铺满风格。

## 口播布局

竖屏 1080×1920 分两大区：

```
┌────────────────────┐
│   ● ● ●  终端状态栏   │  80px HUD 头栏
│   $ 7fit decode     │
├────────────────────┤
│                    │
│   ░░ 信息填充区 ░░   │  主信息区（卡片/数据/高亮/代码）
│   ┌──  ──  ──  ┐  │  占画面约 65%
│   │ 信息元件      │  │  所有视觉信息都在这里
│   └──  ──  ──  ┘  │
│                    │
├────────┬───────────┤
│ 口播框  │ ░░ 信息 ░░ │  右下分区
│ 正方形  │ 辅助文字   │  口播正方形 280×280
│ 280²   │ / 数据  │  右侧剩余空间继续塞信息
└────────┴───────────┘
```

**口播视频位置**：右下角，280×280 正方形容器，`object-fit: cover`。
口播贯穿全片——观众始终能看到"谁在说"。

## 信息密度设计原则

| 原则 | 说明 | 示例 |
|---|---|---|
| **每段至少 2 个信息元件** | 除了口播框，画面上至少还有 2 个信息元素 | 数据卡片 + 高亮文字 / 金句条 + 数字标签 |
| **信息元件跟随声音内容** | 讲到哪个数字/概念，对应元件就高亮/弹出 | 说"30秒"时表单卡片弹入弹窗 |
| **上下左右填空** | 画面不留大面积空白，全部要塞信息 | HUD 头栏 + 信息区 + 底部状态栏 |
| **口播框不挡关键信息** | 口播在右下固定位，主信息区避开该位置 | 信息元件布置在画面上部和左部 |

## 钩子（0-4s，4s）
**画面**：口播在右下。左上区域终端打字"练了 2 年"→"皮质醇过载"→"2 周睡不着"逐行打出。右上数字卡弹入"2周"橙色大号。上中线一条细时间轴从左到右展开标注Week1→Week5。底部状态栏显示"$ 7fit decode v1.0 | 模式: 解码"
**旁白**："练了 2 年，皮质醇过载，2 周睡不着。"

## 段1：皮质醇过载（4-16s，12s）
**画面**：口播在右下。左上信息卡标题"皮质醇检测报告"，下方一条小折线图（训练量持平线 + RPE上升线，简单示意）。右上大号标签"3-4周 训练损失"。左侧列症状文字列："😴 失眠 / ☁️ 踩棉花 / ⚠️ 皮质醇↑"。每条症状讲到时高亮（橙色边框闪过）。底部状态栏更新为"status: 恢复中 → 轻训 2周"
**旁白**："有阵子状态特别差，晚上睡不着，白天像踩在棉花上。花了快一周才反应过来——皮质醇太高了。训练量没变，身体早就在报警了。又花了两周做轻训和有氧才缓过来。"

## 段2：反思——为什么不知道（16-28s，12s）
**画面**：口播在右下。左侧卡片标题"训练记录耗时"，下方一个表单字段列表逐一弹出：
  - 动作名 ☑ / 重量 ☑ / 组数 ☑ / 次数 ☑ / RPE ☑ / 备注 ☑
  每个字段弹出时高亮橙色。右侧大号数字卡"30s"（橙色，缓慢呼吸动画）。讲到最后一句时，底部弹出高亮条"坚持不了 6 个月"（#FF4500 橙色底，白字）。
**旁白**："事后我在想——为什么身体报警了都不知道？因为没有数据。为什么不记数据？因为太麻烦了。以前每次练完掏出手机点 5-6 下。30 秒一组——坚持不了 6 个月。"

## 段3：产品经理的翻译（28-38s，10s）
**画面**：口播在右下。左侧卡片标题"[PRD] 需求#421 训练数据可追溯"，内容占位文字（PM风格）。右侧卡片标题"[训练日志] 2026.03.15"，内容"卧推 80kg 5x8"。两侧卡片底部有注释条——左"监控做在事前"、右"事后才知道"。讲到最后一句时，底部弹出批注条"等价于产品上线一个月才发现用户不用"（代码注释灰色底 #6A9955）。
**旁白**："做产品的都知道——把监控做在事前，比把复盘做在事后便宜得多。健身最大的成本不是动作不标准，是事后才知道自己哪里错了。这就好像产品上线一个月，才发现功能用户根本不用。"

## 段4：解法——小程序（38-52s，14s）
**画面**：口播在右下。左侧一张小程序对话截图卡片（静态截图即可），对话气泡显示"深蹲100kg 5组每组8个"→AI回复"已保存 ✅"。右下口播框上方紧贴一个亮橙色标签"⏱ 3秒"。底部一条流程箭头"语音 → AI解析 → 结构化数据 ✓"。图片下方文字"自然语言记录"高亮。
**旁白**："所以我干脆自己做了小程序。核心就一件事：让你用自然语言记录训练。说一句'深蹲 100kg 5 组每组 8 个'，AI 自动拆成结构化数据。组数、次数、重量全部自动归档。"

## 段5：三个使用场景（52-70s，18s）
**画面**：口播在右下。上部三个迷你场景卡（从左到右横排，约240px宽每张），当前讲到哪张就高亮放大，其余两张缩小变灰。

**场景1（52-58s）**：卡1标题"📏 围度记录"，内容"今天胸围94 腰围78" → "✓已保存 ✅"，卡边框橙色。
**旁白**："你可以这样用——不只记训练，围度也能记。一句话就存好。"

**场景2（58-64s）**：卡1缩小变灰，卡2高亮。标题"🔍 智能查询"，内容"'这周跑了多少次？' → '本周跑步3次，15km'"，底部小字"+2km vs 上周"。
**旁白**："随时问'这周跑了多少次'——AI直接给结论，不用自己翻历史。"

**场景3（64-70s）**：卡2缩小变灰，卡3高亮。标题"📊 AI周报"，内容"训练量+8% / RPE↑ / 建议轻训" + 下周计划卡片预览。底部小字"基于你的数据定制"。
**旁白**："AI根据你的历史数据自动生成下一步。不是网上搬的模板，是基于你数据定制的。"

## 段6：跟人性做朋友（70-80s，10s）
**画面**：口播在右下。左侧大号数字"30"（灰色，偏左），中间大号数字"3"（橙色，偏右），两个数字之间有"VS"小字。"30"下方文字"6个月"，"3"下方文字"能坚持"（绿色）。讲到最后一句时，底部弹出terminal绿文字"不是习惯问题——是产品设计问题"，字体等宽，有光标闪烁效果。同时"30"上划一条斜线（删除线效果）。
**旁白**："能偷懒的地方我一定偷懒——这不是习惯问题，是产品设计问题。30 秒的记录一定坚持不了 6 个月。但 3 秒一句话，就能坚持。"
