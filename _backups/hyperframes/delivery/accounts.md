# 账号档案（accounts.md）

> 双账号详细档案 + 安全设置
> 状态：✅ M1 锁版
> 更新日期：2026-06-04
> 关联：[[strategy.md]]
>
> **账号哲学**：**主号承担"涨粉+转化"（健身 + 七练），副号承担"信任"（vibe coding + 一人公司）**——0 关联是关键。3 要素：**矩阵隔离**（设备/IP/内容 0 关联）+ **人设统一**（主号=健身博主 / 副号=vibe coder）+ **禁区严守**（副号不展示七练产品）。

---

## 1. 主号（健身博主号）

| 字段 | 内容 |
|---|---|
| 平台 | 抖音（现有账号）|
| 账号 ID | _待填_（填你现在的抖音号）|
| 头像 | _当前头像（保留）_ |
| 账号名 | _当前名字（保留）_ |
| 简介关键词 | "健身 2 年 / 训练数据 / AI 工具" |
| 简介禁词 | ❌ 创业 / 独立 / 产品 / 开发 / OPC / 副业 |
| 内容范围 | **只发 B 和 C 类** |
| 目标粉丝 | 健身中级爱好者（练了半年+，开始有数据焦虑）|
| 扩展计划 | M6 考虑扩展视频号、小红书 |

### 主号人设一句话

> "健身 2 年的中级爱好者，业余时间爱研究训练数据 + 写点小工具"

### 主号内容禁区

- ❌ 任何"创业"相关词
- ❌ "我做了一个产品" / "我做的东西"
- ❌ "做产品" / "做开发" / "程序员"
- ❌ 任何指向副号的引导（评论区、私信、主页 bio）
- ❌ 工作场景（办公室、电脑、代码）——纯健身场景
- ❌ 个人情绪宣泄（情绪留给副号）

### 主号内容允许区

- ✅ 训练动作演示（用户自拍）
- ✅ 训练数据展示（图表/数据）
- ✅ 七练产品功能演示（截图/录屏，**以"工具"角度呈现，不强调"我做"**）
- ✅ 健身知识科普
- ✅ 个人训练反思（"我今天练完发现..."）

|---|---|---|
| 粉丝量 | _待填_ | M1 +100 / M2 +300 |
| 内容垂直度 | 100% 健身 | 维持 |
| 简介禁词 | 0 命中 | 维持 |
| 关联防护 | 0 关联 | 0 关联 |
| 更新频率 | 7 条/月 | M2 8-10 条/月 |

## 2. 副号（vibe coding 一人公司号）

> **方向调整（2026-06-04）**：从"独立开发者/创业反思"改为"vibe coding 一人公司实践者"——更垂直、更易算法识别、与主号距离更远。

| 字段 | 内容 |
|---|---|
| 平台 | 抖音（新开账号）|
| 账号 ID | _待注册_（用新手机号注册）|
| 头像 | ❌ **不是**主号头像——选新头像（建议抽象/产品 logo）|
| 账号名 | _待定_（避开"健身/OPC"字眼，命名风格与主号完全不同）|
| 简介关键词 | "vibe coding / 一人公司 / AI 工具流 / 边工作边做产品" |
| 内容范围 | **只发 A 类**（vibe coding 实践 + 一人公司方法论）|
| 目标粉丝 | 独立开发者 / 想做副业的上班族 / 产品经理 / vibe coder |
| 扩展计划 | M6 考虑视频号（产品经理圈在视频号）|

### 副号注册 checklist（M1 启动前必走）

- [ ] **新手机号**（与主号完全不关联的号码）
- [ ] **新设备**（iPad / 备用手机，不与主号共用）
- [ ] **新 IP 段**（不同 WiFi 下注册）
- [ ] **头像**（与主号 0 相似度，优先抽象/logo）
- [ ] **账号名**（3-5 个备选，避开"健身/OPC/创业"字眼）
- [ ] **简介草稿**（关键词对齐 4 选 1，禁词自检 0 命中）
- [ ] **首次浏览**（注册后先浏览 10-20 分钟其他内容，再考虑发第一条）|

### 副号人设一句话

> "一个边工作边用 vibe coding 搭一人公司的人，把 AI 工具流用到极致"

### 副号内容允许区

- ✅ **Vibe coding 实践**：本项目搭建过程、自动化工具流、提示词分享
- ✅ **AI 工具栈**：用的工具、组合方式、效率提升数据
- ✅ **一人公司方法论**：不融资 / 不雇人 / 不全职 / 月预算 ≤1000 元
- ✅ **个人反思**：用 vibe coding 过程中的踩坑、效率对比、真实体验
- ✅ 偶尔穿插健身（**视角是"产品经理自我训练"**，不是"健身博主"）

### ⚠️ 副号禁区（细化版——vibe coding 项目边界）

> **核心原则**：可以**作为 vibe coding 项目**提及七练；不可以**作为产品**展示七练。

| 类别 | ✅ 允许 | ❌ 禁止 |
|---|---|---|
| **项目身份** | "我在做一个 AI 工具" / "我在做 vibe coding 项目" | "我在做七练" / "我在做健身记录" |
| **功能词** | "AI 工具" / "vibe coding 项目" / "个人项目" | "健身记录" / "AI 私教" / "周报" / "3 秒记录" |
| **截图** | 代码截图、编辑器、终端、工具栈 | 产品 UI / 产品 logo / 营销截图 / 演示录屏 |
| **数字** | "vibe coding 用了 X 小时" / "项目代码 Y 行" | "用户量 Z" / "周报数据 W" |
| **讨论** | "如何设计一个 vibe coding 项目" | "七练怎么用 / 七练有什么功能" |
| **品牌符号** | 通用工具 logo（VSCode / Cursor / Claude） | 七练 logo / 七练品牌色 |

### 副号内容禁区（仍然禁止的）

- ❌ 不展示七练产品 UI / Logo / 营销截图
- ❌ 不引导到七练官网/下载
- ❌ 不出现真名 / 工作单位 / 地理位置
- ❌ 不与主号互关/互评/同 IP 登录
- ❌ 副号内容不出现"健身" / "训练" / "私教" / "周报" 等七练产品功能词

## 3. 副号"零关联"操作清单

### 3.1 注册时

- [ ] 用**全新手机号**注册（不要用主号关联手机）
- [ ] **新设备**登录（不同手机或电脑浏览器）
- [ ] **新 IP 段**（如可能，在不同 WiFi 下注册）
- [ ] 头像/名字/简介与主号**完全无关联**
- [ ] 首次浏览/互动其他内容（不直接跳到主号相关内容）

### 3.2 日常使用

- [ ] 主号和副号**不在同一台手机**切换使用（用 iPad / 备用机）
- [ ] 副号发布前**先正常浏览 10-20 分钟**抖音（模拟真实用户）
- [ ] 副号**不互关**主号
- [ ] 副号**不评论**主号的视频
- [ ] 副号**不点赞**主号的内容

### 3.3 内容安全

- [ ] 副号内容**不出现真名**
- [ ] 副号内容**不出现主号任何标识**（头像片段、名字、ID）
- [ ] 副号内容**不展示七练产品**（避免七练 → 副号 → 主号 反查链）
- [ ] 副号口播**不**用主号录制时的同一段背景音 / 同一场景
- [ ] 副号**不**露出主号账号关注列表

### 3.4 设备/IP 隔离 5 维

| 维度 | 主号 | 副号 |
|---|---|---|
| 手机 | iPhone X | iPad / 备用机 |
| 手机号 | 主号关联 | 全新号 |
| WiFi | 家用 WiFi A | 家用 WiFi B / 4G |
| 浏览器 | 日常 | 副号专用 |
| 邮箱 | 主邮箱 | 副邮箱（不互通）|

## 5. 账号健康监控

### 每周自查

- [ ] 副号主页"关注列表"和"粉丝列表"没有出现主号 ID
- [ ] 副号没有被平台标记"异常登录"
- [ ] 两个号的 IP/设备日志没有交叉
- [ ] 副号内容评论区没有主号的"小号"评论

### 每月复盘

- [ ] 副号涨粉曲线（健康吗？有无异常爆涨/爆跌？）
- [ ] 副号"被搜索"来源（自然搜索 vs 关联搜索？）
- [ ] 副号"主页访客"来源（是否有关联账号的访客？）

### 5.1 监控指标速查

| 指标 | 健康范围 | 异常信号 |
|---|---|---|
| 副号日涨粉 | 1-10 | 1 天 100+（爆涨 = 异常）|
| 副号主页访客 | 自然增长 | 集中主号粉丝来访 |
| 副号搜索来源 | 关键词搜索 | 关联账号跳转 |
| 副号被搜索词 | vibe coding / 一人公司 | 健身 / 七练 |

## 6. 紧急预案

### 如果副号被熟人发现

1. **不要慌**——OPC 身份本身不是问题，只是你不想主动告知
2. **统一口径**：副号是"业余兴趣号" / "学习记录号"
3. **评估影响**：发现的是否是关键熟人？影响业务？
4. **是否合并**：如果风险升级，考虑把副号转给可信任的合伙人/朋友运营

### 如果主号被算法误判为"营销号"

1. 检查近 30 天是否频繁发 C 类（产品演示）
2. 暂停 1 周发 C 类，多发 B 类（知识）
3. 检查是否触发了抖音的"广告营销"规则

### 如果副号被限流/封号

1. 立即检查内容是否触发抖音公约（参考 [copy.md §12 违禁词](../planning/copy.md#12-违禁词清单13-类高优处理)）
2. 申诉（如果认为是误判）
3. 备份内容，重新开号（不与主号任何关联）

### 6.1 紧急决策表

| 情况 | 优先级 | 响应时间 |
|---|---|---|
| **副号被熟人发现** | 🔴 高 | 24h 内 |
| **主号被误判营销号** | 🟡 中 | 48h 内 |
| **副号被限流/封号** | 🔴 高 | 24h 内 |
| **主号内容撞车** | 🟢 低 | 1 周内 |
| **副号涨粉异常** | 🟡 中 | 7d 内查 |

## 9 · 反模式

- ❌ 副号用主号手机登录（关联暴露）
- ❌ 副号内容出现"健身 / 七练" 等主号关键词
- ❌ 副号用主号录制时的同一段背景音
- ❌ 副号评论/点赞主号内容
- ❌ 副号简介出现"OPC / 创业" 等反查关键词
- ❌ 副号不监控（涨粉异常/主页访客异常不查）
- ❌ 紧急情况不响应（24h 内无动作）
- ❌ **主号简介出现禁词**（违反 §1）
- ❌ **副号内容出现主号任何标识**
- ❌ **决策不写进 §7 记录**
- ❌ **跳过 5 维评分卡**
