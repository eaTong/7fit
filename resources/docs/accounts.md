## 1. 主号（健身博主号）

| 字段 | 内容 |
|---|---|
| 平台 | 抖音（现有账号）|
| 账号 ID | _待填_（填你现在的抖音号）|
| 头像 | _当前头像（保留）_ |
| 账号名 | _当前名字（保留）_ |
| 简介关键词 | "健身 2 年 / 训练数据 / AI 工具" |
| 简介禁词 | ❌ 创业 / 独立 / 产品 / 开发 / OPC / 副业 |
| 内容范围 | **只发 B 和 C 类** |
| 目标粉丝 | 健身中级爱好者（练了半年+，开始有数据焦虑）|
| 平台 | 抖音（计划 M6 扩展视频号、小红书）|

### 主号人设一句话

> "健身 2 年的中级爱好者，业余时间爱研究训练数据 + 写点小工具"

### 主号内容禁区

- ❌ 任何"创业"相关词
- ❌ "我做了一个产品" / "我做的东西"
- ❌ "做产品" / "做开发" / "程序员"
- ❌ 任何指向副号的引导（评论区、私信、主页 bio）
- ❌ 工作场景（办公室、电脑、代码）——纯健身场景
- ❌ 个人情绪宣泄（情绪留给副号）

### 主号内容允许区

- ✅ 训练动作演示（用户自拍）
- ✅ 训练数据展示（图表/数据）
- ✅ 七练产品功能演示（截图/录屏，**以"工具"角度呈现，不强调"我做"**）
- ✅ 健身知识科普
- ✅ 个人训练反思（"我今天练完发现..."）

## 2. 副号（独立开发者号）

| 字段 | 内容 |
|---|---|
| 平台 | 抖音（新开账号）|
| 账号 ID | _待填_（用新手机号注册）|
| 头像 | ❌ **不是**主号头像——选新头像（建议抽象/产品 logo）|
| 账号名 | _待定_（避开"健身"字眼，命名风格与主号完全不同）|
| 简介关键词 | "独立开发者 / 边工作边做产品 / 健身改造产品" |
| 内容范围 | **只发 A 类**（个人人设/OPC）|
| 目标粉丝 | 独立开发者、产品经理、有副业想法的上班族 |
| 平台 | 抖音（M6 考虑视频号——产品经理圈在视频号）|

### 副号人设一句话

> "一个边工作边做产品的人，把 2 年健身经验当产品做，又用做产品的方式改健身"

### 副号内容允许区

- ✅ 创业故事 / 踩坑反思
- ✅ 产品思维（如何用产品经理视角做副业）
- ✅ 健身改造产品（"我训练时发现 X 痛点，所以做了 Y"）
- ✅ 个人反思 / 时间管理 / 副业经验
- ✅ 偶尔穿插健身动作（但**视角是"产品经理自我训练"**，不是"健身博主")

### 副号内容禁区

- ❌ 直接展示七练产品（避免被反查到主号 → 反查到七练）
- ❌ 七练 logo / 七练产品截图（副号做"创业故事"人设，不是"产品推广"）
- ❌ 真名 / 工作单位 / 地理位置
- ❌ 与主号互动的痕迹（评论主号、点赞主号）

## 3. 副号"零关联"操作清单

### 3.1 注册时

- [ ] 用**全新手机号**注册（不要用主号关联手机）
- [ ] **新设备**登录（不同手机或电脑浏览器）
- [ ] **新 IP 段**（如可能，在不同 WiFi 下注册）
- [ ] 头像/名字/简介与主号**完全无关联**
- [ ] 首次浏览/互动其他内容（不直接跳到主号相关内容）

### 3.2 日常使用

- [ ] 主号和副号**不在同一台手机**切换使用（用 iPad / 备用机）
- [ ] 副号发布前**先正常浏览 10-20 分钟**抖音（模拟真实用户）
- [ ] 副号**不互关**主号
- [ ] 副号**不评论**主号的视频
- [ ] 副号**不点赞**主号的内容

### 3.3 内容安全

- [ ] 副号内容**不出现真名**
- [ ] 副号内容**不出现主号任何标识**（头像片段、名字、ID）
- [ ] 副号内容**不展示七练产品**（避免七练 → 副号 → 主号 反查链）
- [ ] 副号口播**不**用主号录制时的同一段背景音 / 同一场景
- [ ] 副号**不**露出主号账号关注列表

## 4. 内容安全等级评估

| 维度 | 主号 | 副号 |
|---|---|---|
| 暴露风险 | 低（熟人看到=健身博主）| **高**（如果泄露=熟人知道 OPC 身份）|
| 隐私保护 | 常规 | **严格**（所有以上清单）|
| 内容审核强度 | 常规 | **高**（每条发布前自检）|
| 内容备份 | 本地 | **本地 + 加密云备份** |

## 5. 账号健康监控

### 每周自查

- [ ] 副号主页"关注列表"和"粉丝列表"没有出现主号 ID
- [ ] 副号没有被平台标记"异常登录"
- [ ] 两个号的 IP/设备日志没有交叉
- [ ] 副号内容评论区没有主号的"小号"评论

### 每月复盘

- [ ] 副号涨粉曲线（健康吗？有无异常爆涨/爆跌？）
- [ ] 副号"被搜索"来源（自然搜索 vs 关联搜索？）
- [ ] 副号"主页访客"来源（是否有关联账号的访客？）

## 6. 紧急预案

### 如果副号被熟人发现

1. **不要慌**——OPC 身份本身不是问题，只是你不想主动告知
2. **统一口径**：副号是"业余兴趣号" / "学习记录号"
3. **评估影响**：发现的是否是关键熟人？影响业务？
4. **是否合并**：如果风险升级，考虑把副号转给可信任的合伙人/朋友运营

### 如果主号被算法误判为"营销号"

1. 检查近 30 天是否频繁发 C 类（产品演示）
2. 暂停 1 周发 C 类，多发 B 类（知识）
3. 检查是否触发了抖音的"广告营销"规则

### 如果副号被限流/封号

1. 立即检查内容是否触发抖音公约（参考 [copy.md 第 4 节](../remotion/rules/copy.md)）
2. 申诉（如果认为是误判）
3. 备份内容，重新开号（不与主号任何关联）
