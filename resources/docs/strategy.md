## 3. 三类视频占比（M1 试水期）

| 账号 | 类型 | M1 产能 | M2+ 目标产能 | 占比 |
|---|---|---|---|---|
| 主号 | B 健身知识 | 4 条/月（1/周）| 5-6 条/月（1.5/周）| 主号 60-65% |
| 主号 | C 七练介绍 | 3 条/月（隔周 1）| 3-4 条/月（1/周）| 主号 35-40% |
| 副号 | A 个人人设 | 2 条/月（双周 1）| 4 条/月（1/周）| 副号 100% |
| **总计** | — | **9 条/月** | **12-14 条/月** | — |

### 为什么这个比例

| 类 | 角色 | 频率决策 | 风险 |
|---|---|---|---|
| **B 流量款** | 涨粉入口 | 多 | 缺 B 类 = 没新粉 |
| **C 转化款** | 让粉丝知道产品 | 中 | C 太多 = 变硬广 |
| **A 信任款** | 副号独占 | 少 | 副号必须稳节奏，不能多 |

### B 类内容的三层金字塔

**B 类内部优先级**（从高到低）：

```
        🔴 高优先级（每月 3-4 条）
        ━━━━━━━━━━━━━━━━━━━━━━
        「问题导向」内容
        体态问题 + 训练动作问题
        - 高搜索量（SEO 友好）
        - 高互动（评论"我也是"）
        - 高收藏（用户想存）
        - 高完播（自测 + 改善 + 总结）
                ┃
        ━━━━━━━━━━━━━━━━━━━━━━
        🟡 中优先级（每月 1-2 条）
        ━━━━━━━━━━━━━━━━━━━━━━
        「功能演示」内容
        七练功能 + 用户视角
                ┃
        ━━━━━━━━━━━━━━━━━━━━━━
        🟢 低优先级（每月 0-1 条）
        ━━━━━━━━━━━━━━━━━━━━━━
        「训练知识」内容
        营养学/RPE/容量等通识
```

**问题导向为什么是最高优先级**：

| 维度 | 问题导向 | 功能演示 | 训练知识 |
|---|---|---|---|
| 搜索量 | 🔴 高 | 🟡 中 | 🟡 中 |
| 互动 | 🔴 高（"我也是"）| 🟡 中 | 🟢 低 |
| 收藏 | 🔴 高（用户想自测/复习）| 🟡 中 | 🟡 中 |
| 完播率 | 🔴 高 | 🟡 中 | 🟢 低 |
| 涨粉 | 🔴 高 | 🟡 中 | 🟢 低 |
| 转化 | 🟡 中（顺便提七练）| 🔴 高 | 🟢 低 |

**结论**：问题导向是"涨粉 + 互动 + 留存"的综合最优解；功能演示另开 C 类承接。**B 类主推问题导向**。

### 副号 A 类内容策略（vibe coding + 一人公司）

**核心方向**："我用 vibe coding 实现一人公司的过程和理念"——内容**严格围绕 3 个子主题**：

```
        🔴 高优先级（每月 2-3 条）
        ━━━━━━━━━━━━━━━━━━━━━━
        vibe coding 实践
        - 本项目搭建过程（vibe coding 全过程）
        - 自动化剪辑视频探索（AI 工具流）
        - 工具栈 + 提示词分享
                ┃
        ━━━━━━━━━━━━━━━━━━━━━━
        🟡 中优先级（每月 1-2 条）
        ━━━━━━━━━━━━━━━━━━━━━━
        一人公司方法论
        - 不融资/不雇人/不做全职
        - 月预算 ≤1000 元
        - 单人运营 SOP
                ┃
        ━━━━━━━━━━━━━━━━━━━━━━
        🟢 低优先级（每月 0-1 条）
        ━━━━━━━━━━━━━━━━━━━━━━
        vibe coding 工具/提示词分享
        - 提示词模板
        - 工具组合
```

**为什么聚焦"vibe coding + 一人公司"**：

| 维度 | 价值 |
|---|---|
| 内容垂直度 | 极高（新兴垂直赛道） |
| 算法识别 | 容易打标签（vibe coding / 一人公司） |
| 受众精准 | 想做副业 / 独立开发者 / 产品经理 |
| 内容可复制 | 每次项目进展都是新内容（迭代式） |
| 与七练的距离 | **足够远**（讲过程/方法论，不讲产品/营销） |

**禁区**（详见 [[accounts.md]]）：
- ❌ 不展示七练产品 UI / Logo / 营销截图
- ❌ 不引导到七练官网/下载
- ✅ 可以**作为 vibe coding 项目**提及（"我在做 AI 工具"）
- ✅ 可以分享代码、提示词、工具栈

## 4. 每周排期（M1 试水）

```
周日 20:00  ━━  副号  →  A 类（每双周一次，M1 共 2 条）
周二 19:30  ━━  主号  →  B 类（M1 共 4 条）
周五 19:30  ━━  主号  →  C 类（M1 共 3 条）
```

**为什么这样排**：
- 周日 20:00：副号深度反思（用户有空、平台晚高峰）
- 周二 19:30：主号工作日晚流量（小高峰）
- 周五 19:30：主号周末前晚高峰（最大流量）

**M1 完整排期**（4 周 = 9 条）：

| 周 | 周二 19:30 主号 | 周五 19:30 主号 | 周日 20:00 副号 |
|---|---|---|---|
| W1 | B1 | C1 | — |
| W2 | B2 | C2 | A1 |
| W3 | B3 | C3 | — |
| W4 | B4 | — | A2 |

### M2 满产能（12-13 条）

| 周 | 周二主号 B | 周五主号 C | 周日副号 A（vibe coding + 一人公司）|
|---|---|---|---|

## 5. 产能 vs 时间预算

| 阶段 | 周产 | 单条时长 | 周制作小时 |
|---|---|---|---|
| M1 主号 B | 1 | 2h | 2h |
| M1 主号 C | 0.75 | 3h | 2.3h |
| M1 副号 A | 0.5 | 2.5h | 1.3h |
| **M1 周总** | **2.25** | — | **~6h/周** |

**~24h/月** 制作时间 + 4h/月 发布/复盘 = **28h/月**

在 OPC 月预算（≤1000 元 / 单人运营 / 本职工作优先）下**完全可行**。

### M2 满产能时间预算

- ~8h/周 = **32h/月** 制作
- 本职工作优先，单人运营可控

## 6. 起步期（M1）的目标

| 维度 | 目标 |
|---|---|
| 流程跑通 | copy → storyboard → assets → render 全链路验证 |
| 跑通双账号 | 主号发 7 条不踩雷；副号发 2 条不被熟人发现 |
| 数据基线 | 记录每条播放/点赞/收藏/评论 → 知道哪些类型跑得好 |
| 工具稳定性 | mmx TTS / mmx 图片生成 / ffmpeg / Studio 都跑通 |
| 内容模板沉淀 | A/B/C 各产出 2-3 个"模板化"内容，可复用 |

### M1 结束后（M2 启动前）的复盘维度

- [ ] 哪类内容数据最好？（B 应该是流量冠军）
- [ ] 哪类最难做？（C 类可能最耗时间）
- [ ] 副号是否被发现？（如发现 → 调整人设）
- [ ] 制作流程的瓶颈在哪？（可能是素材/字幕/渲染）
- [ ] 是否需要调整 M2 比例？

## 8. 风险与对冲

| 风险 | 概率 | 对冲 |
|---|---|---|
| 副号起号慢（3 个月 0-50 粉）| 高 | 接受"长期主义"——前 3 个月副号当"自我记录" |

| 没素材（B 类动作视频）| 中 | M1 主要靠 B 类已有素材 + 文字数据图 |
| 算法限流 | 低 | 不违规（详见 copy.md 违禁词 + 抖音/小红书公约）|
