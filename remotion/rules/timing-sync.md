# 统一语速控制（timing-sync.md）

> **核心铁律**：视频**所有时间字段是相互推导的，不是独立的**。改一个，必须改所有。
>
> 本规则是 **被 8 个 rules 引用**的：copy / subtitle / storyboard / bgm / script / animation / render / checklist 顶部都加了"必须遵循 timing-sync.md"提示。
>
> **语速哲学**：**主体 50s = 单一锚点**——所有时间字段从它推导。3 要素：**锚点恒定**（50s 不动）+ **4 档速度档位**（场景选档）+ **10 文件同步**（改一全改）。

---

## ⏰ 1 · 单一锚点：主体时长 = 50 秒（用户硬约束）

其他时间字段都从这个锚点推导：

```
中速（默认）   = 主体字数 / 主体时长    → 当前 170 字 / 50s = 3.4 字/秒
钩子            = 3 秒（抖音硬约束）
收尾            = 7 秒
段间停顿        = 0.5-1 秒（用户硬约束）
全文时长        = 钩子 + 主体 + 段间停顿×(N-1) + 收尾  → > 60 秒（N 段主体 = N-1 个停顿）
BGM            ≥ 全文 + 3s fade out    → > 63 秒
Timeline        duration = 全文 × 1    → > 60 秒
```

> **2026-06-07 修订**：原文"全文 = 60 秒"是单段无停顿的最小值，**实际 N 段主体 = N-1 个停顿**会让全文超过 60s。3 段主体 → 2 个停顿（0.7s×2=1.4s）→ 全文 61.4s。**以 `> 60s` 为目标**，不要把 60s 写成硬约束。

### 1.1 锚点推导链

```
主体字数（170）
  ↓ ÷ 3.4
主体时长（50s）⭐ 锚点
  ↓
全文时长（> 60s，含钩子 3s + 段间停顿 + 收尾 7s）
  ↓
BGM 时长（≥ 全文 + 3s fade out = > 63s）
  ↓
VIDEO_DURATION（= 全文，tl.duration() 锚点）
  ↓
字幕总时长（= VIDEO_DURATION，subtitles.json）
  ↓
分镜总时长（= VIDEO_DURATION，storyboard.json）
```

## 2 · 4 档速度档位（提词器顶部选）

| 档位 | 字/秒 | 主体 50s 配套 | 全文 |
|---|---|---|---|
| 🐢 慢 | 2.5 | 68s | > 78s |
| 🚶 **中** ⭐ | **3.4** | **50s** | **> 60s** |
| 🏃 快 | 4.0 | 42s | > 52s |
| ⚡ 原速（不推荐）| 4.8 | 35s | > 45s |

> **默认中速**。用户录制时选其他档位需在 `copy.md` 头部标注。

### 2.1 4 档选型决策

| 场景 | 推荐档位 | 原因 |
|---|---|---|
| **健身知识（B 类）** | 🚶 中速 3.4 | 默认档，知识 + 节奏平衡 |
| **个人 vlog（A 类）** | 🐢 慢 2.5 | 慢节奏带情绪 |
| **动作演示（B 类紧凑）** | 🏃 快 4.0 | 节奏紧凑 |
| **快剪/跳镜（C 类）** | ⚡ 原速 4.8 | 4 镜以内高密度 |
| **老年群体 / 小红书泛流量** | 🐢 慢 2.5 | 让人停下来听 |

### 2.2 档位切换的影响

> **改档位 = 改全文时长 = 改 BGM = 改 VIDEO_DURATION**——4 处必同步。

```
中速 → 慢：全文 60s → 78s
  ↓
- BGM 文件需重选 ≥ 81s
- VIDEO_DURATION 改 78
- 字幕总时长改 78
- 分镜总时长改 78
```

## 3 · 改任何时间字段的同步清单（必走，否则下游会错）

> 改一个时间字段，**必须**同步修改以下 10 个文件中的相关位置：

1. `resources/docs/copy/<主题>.md` — 主体/钩子/收尾/预计时长
2. `resources/docs/copy/<主题>.copy_notes.md` — 字数核对
3. `tools/recording-teleprompter.html` — `DEFAULT_TARGET_SPEED` + 3 档按钮
4. `resources/audios/bgm/*.mp3` — 时长 ≥ 全文 + 3s
5. `hyperframe/src/scenes/<主题>/storyboard.md` — 镜头时长
6. `hyperframe/src/scenes/<主题>/subtitles.json` — 每条 `end - start`
7. `hyperframe/src/scenes/<主题>/scene.js` — `tl.duration()` = VIDEO_DURATION
8. `hyperframe/src/scenes/<主题>/assets.md` — §0 目标时长
9. **`CLAUDE.md`（仓库根）** — 统一语速控制小节
10. **`./timing-sync.md`（本文件）** — §0 默认值表

### 3.1 同步 SOP

```
1. 改 1 个时间字段（如钩子 3s → 5s）
   ↓
2. 推导出新全文时长（> 60s 仍是目标）
   ↓
3. 依次改 10 个文件
   ↓
4. 重跑 [checklist.md §7 跨文件影响图](../delivery/checklist.md#7-跨文件影响图)
   ↓
5. 用户复阅
```

### 3.2 同步顺序建议

> **按"上游→下游"顺序**——文案稿先改，下游 scene/字幕最后。

```
1. copy.md（源）
2. copy_notes.md（核对）
3. recording-teleprompter.html（提词器）
4. assets.md（目标时长）
5. storyboard.md（镜头时长）
6. subtitles.json（每条 end-start）
7. scene.js（tl.duration）
8. bgm 文件（长度 ≥ 新全文 + 3s）
9. CLAUDE.md（默认值表）
10. timing-sync.md（本文件）
```

## 4 · 段间停顿规范（0.5-1s，用户硬约束）

> **目的**：让观众消化前段内容。`pause_breath` shot 必须延长上一个视频的播放时间。

| 段数 | 停顿次数 | 推荐停顿 |
|---|---|---|
| 2 段主体 | 1 | 0.7s |
| 3 段主体 | 2 | 0.5-0.7s × 2 |
| 4+ 段主体 | 3+ | 0.5s × N |

**停顿的视觉实现**（必选其一）：
- 0.8× 慢动作
- 1.2× 加速
- 特写（zoom in 1.2×）
- freeze frame（定格）

**禁止**的停顿实现：
- ❌ 切换其他素材
- ❌ 显示纯字幕
- ❌ 显示装饰卡片

### 4.1 pause_breath 决策表

| 段间位置 | 推荐停顿 | 实现 |
|---|---|---|
| **钩子 → 段 1** | 0.3s（弱停顿）| fade / 文字残影 |
| **自测 → 动作** | 1.0s（重点消化）| frozen frame + 0.8× 慢动作 |
| **动作 → 动作** | 0.5s（紧凑）| quick cut / push_left |
| **主体 → 收尾** | 1.0s（沉淀）| fade out + zoom out |
| **收尾 → CTA** | 0.3s | 文字淡入 |

> 详见 [script.md §5.2 pause_breath 4 种实现](../production/script.md#52-pause_breath-4-种实现)。

### 4.2 破折号停顿规范（v4 新增，2026-06-09）

> **来源**：`gym_machine_judge_b13` 实战（段 1 用 12 个破折号做内部停顿标记）

| 停顿类型 | 时长 | 数量约束 | 适用 |
|---|---|---|---|
| **标点**（"。" "？" "；"） | 0.3s | 自然 | 所有视频 |
| **强调破折号**（"——"） | 0.3-0.5s | 一段最多 1 个 | 钩子/CTA 关键停顿 |
| **内部停顿标记**（"——"） | **≥ 0.5s** | 可每句 2-3 个 | 通用方法论/模式识别类视频的视觉演示 |
| **段间停顿** | 0.5-1s | 0.5-0.7s × N | 段落切换 |

**内部停顿标记** 适用场景（B 类"通用方法论"）：
- 4 步法总段的多器械枚举演示（接触点 + 调节）
- 每说 1 个"这里/这样"占位符时，video 切 1 个不同器械的镜头
- 破折号创造 0.5s 视觉切换停顿，让观众有时间"看到"每个器械
- 段 2/段 3（动作指令密集）保持无破折号，避免打断动作节奏

**计算公式**（同步 §6.1 VIDEO_DURATION）：
```js
// 句子 duration = speech 字数 / 速度 + 破折号数 × 0.5
sentenceDuration = speechChars / speed + dashCount * 0.5
```

**为什么是 0.5s 不是 0.3s**：
- 标点 0.3s = 自然换气停顿（足够理解文字）
- 破折号 0.5s = 视觉切换停顿（足够 video 切镜 + 看清器械）
- **停顿时间 ≥ 视频切镜的最小时间**（< 0.5s 视频切换显得仓促）

**与 [copy.md §5.3.1](../../planning/copy.md#531-破折号-2-种模式v4-新增) 同步**——两种模式（强调 vs 内部停顿）配合使用。

## 5 · 时长计算示例

### 5.1 3 段主体、170 字、中速

```
钩子            =  3.0s
段 1            =  6.0s（30 字 / 3.4 ≈ 8.8s）实际 8.8s
停顿            =  0.7s
段 2            =  8.0s（40 字 / 3.4 ≈ 11.8s）实际 11.8s
停顿            =  0.7s
段 3            =  7.0s（50 字 / 3.4 ≈ 14.7s）实际 14.7s
收尾            =  7.0s
─────────────────────────────────
全文时长        =  3 + 8.8 + 0.7 + 11.8 + 0.7 + 14.7 + 7 ≈ 46.7s

> 但实际还要看钩子/收尾的展开时长，所以最终全文 > 50s
```

### 5.2 校验公式

```js
// 主体总字数 = 钩子字数 + 主体字数 + 收尾字数
// 主体总时长 = 主体字数 / 中速
// 全文时长 = 主体总字数 / 中速 + 段间停顿 × (N - 1) + 收尾展开
// 全文时长 > 60s（不是 = 60s）
// BGM 时长 ≥ 全文时长 + 3s
```

### 5.3 时长反推（按段数）

| 段数 | 段主体字数（参考）| 主体时长 | 段间停顿 | 全文估算 |
|---|---|---|---|---|
| 2 段 | 30 + 50 = 80 | 23.5s | 0.7s | ~33.2s（含钩子+收尾）|
| 3 段 | 30 + 40 + 50 = 120 | 35.3s | 1.4s | ~46.7s |
| 4 段 | 30 + 40 + 40 + 60 = 170 | 50s | 1.5s | ~61.5s |
| 5 段 | 30 + 30 + 40 + 30 + 40 = 170 | 50s | 2.0s | ~62.0s |

> **170 字 / 中速 / 4 段 = 主体 50s + 全文 61.5s** = 标准 60s+ 视频。

## 6 · 单条字幕时长上限（单源真相）

> **本节是单条字幕的硬上限** —— 无论用哪档速度，每条字幕都受此约束。subtitle.md §6 引用本节，避免双向重复。

### 6.1 单条时长上限

- **单条 ≤ 4 秒**（用户阅读时间，超出走神）
- **单条 ≤ 24 字**（视觉密度）

> **超出**：必须拆成多条。详 [subtitle.md §8 拆条与合并策略](../production/subtitle.md#8-拆条与合并策略)。

### 6.2 公式校验

```js
// 主体区段：每条字幕的 (end - start) ≈ text.length / 3.4
// 例：6 字文本 → 6 / 3.4 ≈ 1.76s → 实际 1.7-1.8s 合理
// 例：24 字文本 → 24 / 3.4 ≈ 7.06s → 实际 7.0-7.1s 合理
// 允许 ± 0.3s 误差
```

## 7 · 时长异常处理

### 7.1 异常场景 + 修法

| 异常 | 现象 | 修法 |
|---|---|---|
| **全文 < 60s** | 视频太短 | 加段数 / 延长收尾 / 加 pause_breath |
| **全文 > 90s** | 视频太长 | 拆条发布（[publish.md §1](../delivery/publish.md#1-发布前准备)）/ 减段数 |
| **BGM < 全文** | 末段静音 | 重选 BGM（[bgm.md §1.1](../production/bgm.md#11-bgm-长度公式)）|
| **字幕单条 > 4s** | 视觉过密 | 拆条（[subtitle.md §8](../production/subtitle.md#8-拆条与合并策略)）|
| **字幕单条 > 24 字** | 视觉过密 | 拆条 |
| **VIDEO_DURATION 与字幕不同步** | 末段错位 | 改 `tl.duration()`（[render.md §6](../delivery/render.md#6-timeline-duration-硬约束)）|

### 7.2 异常处理 SOP

```
1. 跑 §7.1 异常场景检查
   ↓
2. 定位根因（哪个字段错）
   ↓
3. 跑 §3 同步清单（10 文件）
   ↓
4. 重跑 [checklist.md §6.3 渲染前 50+ 项](../delivery/checklist.md#6--场景化自检3-场景--必跑项)
   ↓
5. 重新渲染
```

## 8 · VIDEO_DURATION 全链路同步表

> **VIDEO_DURATION 是单源真相**——`scene.js` 中的常量。所有下游字段必须与它一致。

| 字段 | 来源 | 同步位置 |
|---|---|---|
| **VIDEO_DURATION** | `timing-sync.md` 推导 | `scene.js` 顶部常量 |
| **timeline.duration** | = VIDEO_DURATION | `scene.js` |
| **subtitles 总时长** | = VIDEO_DURATION | `subtitles.json` 末条 `end` |
| **storyboard 总时长** | = VIDEO_DURATION | `storyboard.json` 末镜 `end` |
| **bgm 文件时长** | ≥ VIDEO_DURATION + 3s | `resources/audios/bgm/` |
| **渲染命令** | `--duration=VIDEO_DURATION` | `npm run render` |

> 改 VIDEO_DURATION = 6 处必同步。

## 9 · 5 维评分卡（时间字段一致性）

> **每维 ≥ 3 分才能进渲染**，总分 ≥ 15/25。

| 维度 | 1 分（差）| 3 分（中）| 5 分（优）| 本稿得分 |
|---|---|---|---|---|
| **锚点合规** | 主体 ≠ 50s / 中速 ≠ 3.4 | 主体 50s + 中速 3.4 | 主体 50s + 中速 3.4 + 写进 copy.md 头部 | — |
| **时长连贯** | 全文 < 60s / BGM 短 | 全文 > 60s + BGM 够 | 全文 > 60s + BGM ≥ 全文+3s + 6 处同步 | — |
| **段间停顿** | 无停顿 / 用错实现 | 0.5-1s + 4 选 1 | 0.5-1s + 4 选 1 + 决策表选对 | — |
| **字幕合规** | > 30% 字幕违 4s/24 字 | < 10% 违反 | 0 违反（自动校验过）| — |
| **跨文件同步** | 改 1 个不同步 | 改 1 个同步 3-4 处 | 改 1 个全 6 处同步 + 走 §3 清单 | — |

## 10 · 反模式

- ❌ 改一个时间字段不同步其他 9 个文件（"应该没问题"就是问题）
- ❌ 全文 = 60s 写成硬约束（实际 N 段主体 = N-1 个停顿 → > 60s）
- ❌ 段间停顿用切换其他素材 / 纯字幕 / 装饰卡片（破坏消化节奏）
- ❌ BGM 比全文短（视频末尾静音）
- ❌ 录完旁白才发现节奏不对，回头改文案但不改分镜/字幕
- ❌ 把 4 档速度档位的默认值偷偷改掉（不通知用户）
- ❌ **改 VIDEO_DURATION 不同步 6 处**（详 §8）
- ❌ **不跑异常处理表**（详 §7.1）
- ❌ **段间停顿不在 0.5-1s 范围**
- ❌ **跨文件影响图不重跑**（违反 [checklist.md §7](../delivery/checklist.md#7-跨文件影响图)）
- ❌ **跳过 5 维评分卡直接渲染**
