# 跨阶段时间同步规则（remotion）

> 阶段：**贯穿内容创作 → 录音 → 分镜 → 渲染 → 剪辑 全流程**
> 适用场景：调整主体时长 / 调整语速 / 全文时长 / BGM 时长 / 字幕时长 / 镜头时长 —— **任何时间字段改动**
> 来源：用户硬约束（2026-06-04 决策：主体 50s 为锚点）
> 状态：✅ 生效
> 上下游：**被所有时间相关 rules 引用**（copy / subtitle / storyboard / bgm / script / animation / render）
> 位置：`remotion/rules/timing-sync.md`

> **核心原则**：**所有时间字段是相互推导的，不是独立的**。改一个，必须改所有。

---

## 0. 单一锚点：主体 = 50 秒

> 视频所有时间字段**只有一个根**——**主体时长 = 50 秒**。
> 所有其他时间从它推导。改 50s 这一行，其他全部联动。

| 字段 | 来源 | 公式 | 默认值（当前）|
|---|---|---|---|
| **主体时长** ⭐ | **用户硬约束** | 固定 | **50 秒** |
| **主体字数** | copy.md 主体段 | countChars(主体) | 169 字 |
| **中速（默认）** | 主体字数 ÷ 主体时长 | 169 / 50 | **3.4 字/秒** |
| **钩子时长** | 抖音硬约束 | 固定 | 3 秒 |
| **收尾时长** | 抖音规范 | 固定 | 7 秒 |
| **段间停顿** 🆕 | **用户硬约束（2026-06-04）** | 每段落之间 | **0.5-1 秒**（可调 0.5/0.7/1.0） |
| **全文时长** | 钩子 + 主体 + 段间停顿×N + 收尾 | 3 + 50 + N×0.7 + 7 | **~60-65 秒** |
| **BGM 时长** | ≥ 全文 + 2-3s fade out | ≥ 60 + 3 | **≥ 63 秒** |
| **字幕时长** | 字幕条字数 ÷ 中速 | sum(每条) | 60 秒（**不含停顿**）|
| **镜头时长** | 字幕时长 + 段间停顿（含 1:N pack） | sum(shots) + N×停顿 | 全文时长 |
| **Composition durationInFrames** | 全文 × fps | 65 × 30 | **1950 帧**（含停顿） |

> **改 50s 这一行**：所有 10 个字段全部联动。
> 🆕 **段间停顿**也是独立参数——改它 → 全文时长变 → 7 个文件同步（见 §3）。

## 1. 4 档速度档位

> 用户在 `recording-teleprompter.html` 顶部选档位。
> **中速是默认** = 由锚点（主体 50s）自动算出，**不是写死的 4.8**。

| 档位 | 字/秒 | 主体 50s 配套时长 | 适用场景 |
|---|---|---|---|
| 🐢 **慢** | 2.5 | 主体 68s / 全文 **78s** | 第一次念稿 / 数字密集段 / 想保稳 |
| 🚶 **中** ⭐ | **3.4** | 主体 **50s** / 全文 **60s** | 默认 / 正常录制 |
| 🏃 **快** | 4.0 | 主体 42s / 全文 **52s** | 二稿 / 复练 / 节奏感已建立 |
| ⚡ **原速** | 4.8 | 主体 35s / 全文 **45s** | 仅作 v3 旧版本对比用，**不推荐**（中速已是最优）|

> **为什么默认是"中速"而不是"原速"？**
> - 4.8 字/秒 是机器自动切分算出来的"理论最快"
> - 实际说话（朋友聊天感）= 3-3.5 字/秒（含语气词/停顿）
> - 4.8 字/秒 念出来会很"播音腔"，**冲淡七练"同侪陪伴"调性**
> - 中速 3.4 字/秒 = 主体 50s，**留够 2 倍缓冲给停顿/重音**

## 1.5 🆕 段间停顿（用户硬约束 · 2026-06-04）

> **核心**：每个段落（钩子段 / 主体各分段 / 收尾段）之间必须有 **0.5-1 秒的静音停顿**——让观众消化前段内容。
> 这不是"破折号"的语内停顿，是**段间真正的静音**。

### 1.5.1 默认值

| 段间位置 | 默认停顿 | 可选档位 |
|---|---|---|
| 钩子 → 主体第 1 段 | **0.7 秒** | 0.5 / 0.7 / 1.0 |
| 主体段内段落之间（自测 1 → 自测 2 → 4 个动作）| **0.7 秒** | 0.5 / 0.7 / 1.0 |
| 主体最后段 → 收尾 | **0.7 秒** | 0.5 / 0.7 / 1.0 |
| 收尾 → 结束 | 0（无）| — |

> **为什么是 0.7 秒？**
> - 0.5 秒太短，观众没时间消化
> - 1.0 秒太长，节奏会断
> - 0.7 秒 = **黄金中点**，既给观众喘息又维持节奏
> - 抖音/小红书热门视频统计：90% 段间停顿在 0.5-0.8s

### 1.5.2 段间停顿 ≠ 字幕停顿

| 项 | 段间停顿 | 字幕内的 `——` 停顿 |
|---|---|---|
| **作用** | 段落之间静音 | 句子内部重音 / 强调 |
| **持续时间** | 0.5-1.0 秒 | 0.3-0.5 秒 |
| **字幕** | 段间**不放字幕**（黑屏/转场）| 字幕内显示 "⏸ 0.5s" 红块 |
| **BGM** | BGM 在停顿期间升回 -8 dB | BGM 维持 -12 dB（ducking）|

### 1.5.3 段间停顿在 copy.md / 字幕 / 故事板 / BGM 各自怎么落地

| 阶段 | 怎么标停顿 |
|---|---|
| **copy.md** | 段落之间用**单独一行的 `>`** 标记（视觉上是空行，但工具识别为段间停顿）|
| **提词器** | 自动在每个段间加 0.7s 计时（已实现，详见 parseScript）|
| **subtitles.json** | 段间**不放字幕**——前一条 `end` 到下一条 `start` = 0.7s 空白 |
| **storyboard.md** | 段间 shot 是"**空镜/转场镜**"，时长 = 0.7s + 转入/转出动效 |
| **BGM** | 段间停顿期间 BGM 升回 -8 dB（解除 ducking），段内维持 -12 dB |
| **Composition** | durationInFrames = (字幕总时长 + 段间停顿数 × 0.7) × fps |

### 1.5.4 改段间停顿必走的同步清单

> 段间停顿也是时间参数——改它必须 7 个文件同步：

1. `copy.md` — 段间空行 + 注释 `<!-- 段间停顿 0.7s -->`
2. `recording-teleprompter.html` — 段间计时逻辑（如未来加可调）
3. `subtitles.json` — 段间空白时长
4. `storyboard.md` — 段间 shot 数量 + 时长
5. `Composition` — durationInFrames 重算
6. `BGM` — 段间 ducking 解除时长
7. `assets.md` — 段间 shot 用的"空镜/转场"素材（如有）

## 2. 4 档速度对应的"全文时长"（自动重算表）

> 改档位时**所有阶段时间字段自动重算**。本表是单一来源。

| 档位 | 字/秒 | 钩子 3s | 主体 | 段间×3 | 收尾 7s | **全文** | BGM（+3s fade）| 帧数（30fps）|
|---|---|---|---|---|---|---|---|---|
| 慢 2.5 | 2.5 | 3s | 68s | 2.1s | 7s | **80s** | ≥ 83s | 2400 帧 |
| **中 3.4** ⭐ | 3.4 | 3s | **50s** | 2.1s | 7s | **62s** | **≥ 65s** | **1860 帧** |
| 快 4.0 | 4.0 | 3s | 42s | 2.1s | 7s | **54s** | ≥ 57s | 1620 帧 |
| 原速 4.8 | 4.8 | 3s | 35s | 2.1s | 7s | **47s** | ≥ 50s | 1410 帧 |

> 段间×3 = 钩子→主体(0.7) + 主体内(0.7) + 主体→收尾(0.7) = 2.1s。主体有更多段间 → 加。

## 3. 时间同步清单（修改后必须改的 7 个文件）

> ⚠️ **核心铁律**：改主体时长 / 改语速 / 改钩子收尾 → 下面 7 个文件**全部同步**，缺一不可。

| # | 文件 / 字段 | 同步方式 | 不改的后果 |
|---|---|---|---|
| 1 | `resources/docs/copy/<主题>.md`<br>§ 主体标题 + § 元数据「预计时长」| 主体 (XX 秒) + 预计 (XX 秒) | 提词器时长不准 |
| 2 | `resources/docs/copy/<主题>.copy_notes.md`<br>§ 时长与字数核对 | 字数 + 时长 + 中速公式 | 后续作者算错 |
| 3 | `remotion/tools/recording-teleprompter.html`<br>DEFAULT_TARGET_SPEED + 3 档按钮 | 改 3 个数字 | 提词器还按旧速度走 |
| 4 | `resources/audios/bgm/*.mp3`<br>BGM 时长 | 用 mmx 重生 ≥ 全文 + 3s | BGM 比旁白短 → 末段静音 |
| 5 | `remotion/src/scenes/<主题>/storyboard.md`<br>§ 0 总览时长 + 每个 shot 时长 | 镜头时长 = 字幕时长（含 1:N pack）| 视频和旁白对不上 |
| 6 | `remotion/src/scenes/<主题>/subtitles.json`<br>每条 start / end | end - start = 字数 ÷ 中速 | 字幕提前消失/末段静音 |
| 7 | `remotion/src/scenes/<主题>/index.tsx`<br>Composition durationInFrames | fps × 全文 + buffer | 渲染后时长不对 |
| 8 | `remotion/src/scenes/<主题>/assets.md`<br>§ 0 状态总览「目标时长」| = 全文时长 | 状态总览失真 |
| 9 | `CLAUDE.md`（项目根）<br>§ 视频制作总流程 步骤 0 + 步骤 8 | 指向本文件 | 后续 Claude 不知道规则 |

## 4. 各阶段"时间字段"对照表

| 阶段 | 字段 | 公式 | 默认值 |
|---|---|---|---|
| **copy.md** | `**预计时长**` | 钩子 + 主体 + 收尾 | 60s |

## 5. 修改时间字段的 5 步工作流

> 假设要把"主体 50s"改成"主体 60s"（比如要加 1 个新动作）：

```
第 1 步：改 copy.md
   - 主体段新增 1 个动作
   - 重算中速：169 → 200 字 / 60 秒 = 3.3 字/秒
   - 改 预计时长：60s → 70s

第 2 步：改 copy_notes.md
   - § 时长与字数核对 同步更新（字数 + 中速 + 全文）

第 3 步：跑 npm run scenes:regen
   - 重新生成 scenes.json + 提词器 SCENES_DATA

第 4 步：浏览器硬刷
   - 提词器：DEFAULT_TARGET_SPEED 自动随中速更新？
   ❌ NO——HTML 里是写死的常量，要手动改 DEFAULT_TARGET_SPEED 和 3 档按钮

第 5 步：全链路验证
   - 提词器选场景 → 进度列表总时长 ≈ 70s
   - 录旁白 → 时长 ≈ 70s
   - mmx 字幕识别 → subtitles.json 每条 end-start 合理
   - storyboard.md → 每个 shot duration 加起来 = 70s
   - Composition durationInFrames = 2100 (70s × 30fps)
   - BGM ≥ 73s
```

> **未来改进**：把 `DEFAULT_TARGET_SPEED` 改为从 copy.md 自动算（每次 regen 时按中速公式重写 HTML 的 SPEED_OPTIONS）—— 这样改 copy.md 后提词器**自动同步**。但**当前手动同步**。

## 7. 速查清单（本文件专属）

**改主体时长之前**：

- [ ] 想清楚"加内容"还是"加快语速"——前者是改 copy.md，后者是改中速
- [ ] 全文 = 钩子 + 主体 + 收尾 是否仍成立？
- [ ] BGM 长度是否够？
- [ ] 7 个文件同步清单（§3）已检查

**改中速之前**：

- [ ] 主体字数已固定？
- [ ] 用户接受这个新语速？（3.0 以下会很慢，5.0 以上会很播音腔）
- [ ] 提词器 3 档按钮同步改了？

**录完旁白后**：

- [ ] 音频时长 ≈ 全文时长 ± 3 秒
- [ ] 如果差太多（>5 秒），重录或调语速
- [ ] mmx 字幕识别时**不传速度参数**（让它自动按字数切分），但**事后再校准**每条 subtitle 的 end 字段

**渲染前**：

- [ ] Composition durationInFrames = 全文 × fps
- [ ] BGM 总时长 ≥ Composition durationInFrames（否则末段静音）
- [ ] 所有 Sequence premountFor 覆盖完整时长
