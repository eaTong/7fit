# 视频类型规范（remotion）

> 阶段：**视频类型分类（video taxonomy）**——决定整套设计/分镜/素材策略
> 适用场景：用户提出"做一条视频"时，**第一件事就是判断属于 3 类中的哪一类**
> 来源：用户硬约束
> 状态：✅ 生效
> 上下游：上游 = `copy.md`（确定主题后归类）；下游 = 全部其他 rules（不同类型的实现方式不同）

---

## 1. 三类视频速查

| 类型 | 主体 | 辅助 | 典型场景 | 默认 BGM |
|---|---|---|---|---|
| **A. 个人人设（OPC）** | 口播（人脸+语音）| 视频片段 / 关键数据 | 自我介绍、理念分享、反思故事 | Quiet Think（安静、有思考感）|
| **B. 健身知识** | 动作演示（视频/动图）| 关键参数 / 数据 | 动作教学、知识科普、问题解析 | Power Build（有力量感）|
| **C. 七练介绍** | 弹出内容（截图/操作/代码/git log）| 健身视频作为背景 | 产品功能演示、技术分享、操作流程 | Cyber Pulse（科技感）|

> ⚠️ **第一件事**：用户说"做 X 视频"时，先告诉他这是 3 类中的哪一类，得到确认后再进入下一步。

## 2. 类型 A：个人人设（OPC）—— 口播为主型

### 2.1 视觉定义

```
┌──────────────────────────────────────┐
│  [顶部标题区 ≥120px 安全区]            │
│                                      │
│      ┌─────────────┐                 │
│      │             │   [字幕]         │
│      │   人脸/上半身 │   (底部)         │
│      │   (主体)     │                 │
│      │             │                 │
│      └─────────────┘                 │
│                                      │
│  [关键数据/文字 overlay（角落小卡片）]   │
└──────────────────────────────────────┘
```

### 2.2 内容层级

| 层级 | 占比 | 内容 |
|---|---|---|
| 主体 | 60-70% 画面 | 人脸 / 上半身（用户自拍口播）|
| 辅助层 1 | 偶尔切入 5-15s | 视频片段（动作演示、场景画面）|
| 辅助层 2 | 持续 | 关键数据/文字 overlay（小卡片形式）|
| 字幕 | 持续 | 同步显示用户说的话 |

### 2.3 主体内容要求

- **必须用真人**（不能是 AI 数字人）
- 拍摄建议：
  - 机位：手机正面 / 三脚架固定 / 1080p
  - 光线：自然光或环形补光灯，避免顶光
  - 构图：人脸在画面中央或偏左，避开右侧被字幕遮挡
  - 背景：简洁不杂乱（深色墙 / 书架 / 健身房一角）

### 2.4 辅助视频片段（切入口播）

- **每个视频片段 5-15s**，由 **PiP（画中画）或全屏切换**方式呈现
- 进入方式：3 种选择
  - **全屏切换**：口播画面 → 视频 → 切回口播（用于"举个例子"）
  - **画中画**：口播缩小到右下角，左侧/上方展示视频（用于"边讲边演示"）
  - **贴片**：视频作为背景虚化（30% 透明度），前景口播
- **七练默认**：全屏切换（最简洁），偶尔用画中画

### 2.5 关键数据 overlay

- 仅展示**1-2 个关键数字/词**，不堆砌
- 位置：右上角或左下角
- 样式：半透明彩色卡片（参考 `script.md` 第 5.1 节）
- 例：右上角小卡片 `2 年`，左下角 `80KG`

### 2.6 BGM 选型

- **默认 Quiet Think**（ambient electronic, 80 BPM）—— 安静、有思考感
- 不用 Cyber Pulse（太"产品演示"）也不用 Power Build（太"激情")
- BGM 音量按 `bgm.md` 第 3.2 / 3.4 节 ducking 规则：旁白期间 -12 dB，无人声时 -8 dB

### 2.7 时长与结构

| 视频长度 | 结构 |
|---|---|
| **15-30s（短视频）** | 钩子（3s）→ 主题讲述（10-20s）→ CTA（3-5s）|
| **1-3min（中视频）** | 钩子（5s）→ 故事/反思（30-90s）→ 升华/CTA（15-30s）|

### 2.8 典型场景示例

- "我用 2 年时间，从手动记 Excel 到做出七练这个产品"
- "健身这事儿，我曾经走过最大的弯路是..."
- "为什么主理人也是用户？"
- "产品思维用在我自己身上..."

## 3. 类型 B：健身知识 —— 动作展示为主型

### 3.1 视觉定义

```
┌──────────────────────────────────────┐
│  [顶部标题区 ≥120px 安全区]            │
│                                      │
│   ┌────────────────────────────┐     │
│   │                            │     │
│   │     动作演示视频/动图        │     │  ← 主体（70%+ 画面）
│   │     (可能 1-4 个分屏)         │     │
│   │                            │     │
│   └────────────────────────────┘     │
│                                      │
│   [关键参数 overlay 卡片]            │  ← 辅助（关键数据）
│   [字幕]                              │
└──────────────────────────────────────┘
```

### 3.2 内容层级

| 层级 | 占比 | 内容 |
|---|---|---|
| 主体 | 70%+ 画面 | 动作演示视频（用户自拍 / mmx 生成图）|
| 辅助层 1 | 持续 | 关键参数（重量/次数/组数/动作要点）|
| 辅助层 2 | 偶尔 | 数据可视化（折线/柱状/对比图）|
| 字幕 | 持续 | 同步口播 |

### 3.3 主体内容要求

- **必须是真实训练动作**——七练不信任 mmx 生成的"假人健身"（已在 `assets.md` 第 4.2 节说明）
- 拍摄建议：
  - 角度：**正侧面 / 45° 角** 各 1 遍（让观众看清动作轨迹）
  - 时长：每个动作 ≥ 5s（满足 `storyboard.md` 第 3 节"视频 > 5s"硬约束）
  - 速度：正常速度 + 偶尔 0.5x 慢动作（关键发力点）
  - 光线：充足，让肌肉线条可见
- 如果没有用户视频：用图片序列（多个角度的关键帧）+ 慢速循环

### 3.4 关键参数 overlay

**必填**（每个动作都该有）：

| 字段 | 示例 |
|---|---|
| 动作名 | 深蹲 |
| 重量 | 100KG |
| 次数 × 组数 | 8 × 4 |
| RPE/难度 | RPE 8 |

**样式**：右下角或左侧的半透明橙色卡片（参考 `script.md` 第 5.1 节），持续显示不消失

### 3.5 分屏（多角度对比）

可选，**仅在关键动作**用（如"正确 vs 错误"对比）：

```
┌──────────────┬──────────────┐
│  正确姿势     │  错误姿势     │
│  ✓ 橙色描边   │  ✗ 红色描边   │
│  (#FF4500)   │  (#DC143C)   │
│              │              │
└──────────────┴──────────────┘
```

> ⚠️ 用品牌色板的强调色（橙/红）做对比描边，**不要引入绿色等其他颜色**——保持与七练色板一致（详见 `script.md` 第 5 节）。

分屏时视频类分镜时长 ≥ 5s（每个画面都 ≥ 5s，让观众看清）

### 3.6 BGM 选型

- **默认 Power Build**（tech house, 105 BPM）—— 有力量感，与训练内容匹配
- 不需要 Quiet Think（教学视频需要节奏感）
- BGM 音量按 `bgm.md` 第 3.2 / 3.4 节 ducking 规则：旁白期间 -12 dB，无人声时 -8 dB

### 3.7 时长与结构

| 视频长度 | 结构 |
|---|---|
| **15-30s（短视频）** | 1 个动作演示 + 关键参数（10-25s）|
| **1-3min（中视频）** | 主题引入（10s）→ 3-5 个动作演示（每个 20-40s）→ 总结/CTA（10-20s）|

### 3.8 典型场景示例

- "深蹲 100KG 怎么做？3 个关键要点"
- "卧推练胸没感觉？这 2 个细节要改"
- "硬拉容易腰酸？核心是这个"
- "为什么我推荐练背不练手臂"

## 4. 类型 C：七练介绍 —— 弹出内容型

### 4.1 视觉定义

```
┌──────────────────────────────────────┐
│  [顶部标题区 ≥120px 安全区]            │
│                                      │
│   ┌────────────────────────────┐     │
│   │  健身视频背景               │     │  ← 背景层（半透明/模糊/降饱和）
│   │  (持续播放)                 │     │
│   └────────────────────────────┘     │
│   ┌────────────────────────────┐     │
│   │                            │     │
│   │  弹出内容卡片               │     │  ← 主体（截图/操作视频/代码/git log）
│   │  (按场景弹出)               │     │
│   │                            │     │
│   └────────────────────────────┘     │
│                                      │
│   [字幕]                              │
└──────────────────────────────────────┘
```

### 4.2 内容层级

| 层级 | 占比 | 内容 |
|---|---|---|
| 背景层 | 全画面（弱化）| 健身视频循环（用户自拍 / mmx 抽象动画）|
| 主体 | 中心 60% 画面 | 弹出内容卡片（按场景切换）|
| 字幕 | 持续 | 同步旁白 |

### 4.3 背景层要求

- 持续循环（让画面"活着"，不静止）
- **必须弱化处理**（不能让背景抢主体）：
  - 方案 A：半透明覆盖（背景 30-50% 透明度 + 黑色蒙层）
  - 方案 B：模糊滤镜（`backdrop-blur` 或 CSS `filter: blur(8px)`）
  - 方案 C：降饱和 + 调暗（`saturate(0.5) brightness(0.6)`）
- 七练默认：**方案 A + 方案 B 组合**（半透明 + 模糊），效果最好
- 背景视频来源：
  - 用户自拍训练视频（推荐）
  - mmx 生成的抽象动画（粒子、几何）
  - **不用**实拍的产品截图（与"健身"主题不符）

### 4.4 弹出内容类型

| 类型 | 用途 | 实现 |
|---|---|---|
| **截图** | UI 展示、产品功能演示 | `staticFile('images/xxx.png')` + 卡片 |
| **操作视频** | 演示用户操作流程 | `staticFile('videos/xxx.mov')` + 卡片 |
| **代码块** | 技术分享、代码展示 | 文本 + 等宽字体 + 暗色背景 |
| **git log** | 开发过程展示 | 文本 + 时间戳 + commit 摘要 |
| **数据图表** | 数据可视化 | `data_viz` 类型分镜 |
| **产品文案** | Headline + 副标题 | 文字 + 半透明彩色背景 |

### 4.5 弹出内容的视觉样式

**统一卡片规范**（与 `script.md` 第 9 节素材框一致）：

```tsx
// ✅ 弹出卡片：半透明橙色背景 + 橙色描边 + 圆角
<div className="bg-[#FF4500]/15 border-2 border-[#FF4500]/50 rounded-xl p-6 backdrop-blur-sm">
  <h3 className="text-white text-2xl font-bold mb-3">3 秒记录</h3>
  <p className="text-white/80 text-base">对话式记录，AI 自动解析</p>
</div>
```

**进入动效**：从下方浮入 + 弹性入场（参考 `animation.md` 第 5 节）
**退出动效**：向上淡出
**多个卡片切换**：Crossfade 0.3s（参考 `script.md` 第 8 节转场）

### 4.6 卡片切换逻辑

按"什么场景显示什么内容"：

| 场景（旁白在说）| 弹出内容 |
|---|---|
| "3 秒记录训练" | 截图：AI 对话界面 |
| "自动解析动作" | 代码块：解析逻辑 |
| "周报数据复盘" | 数据图表：折线/柱状 |
| "git log 显示开发过程" | git log 列表 |
| "操作流程演示" | 操作视频（嵌入卡片中）|

### 4.7 BGM 选型

- **默认 Cyber Pulse**（synthwave, 100 BPM）—— 科技感，与"产品演示"主题最匹配
- 不需要 Power Build（不强调训练强度）
- BGM 音量按 `bgm.md` 第 3.2 / 3.4 节 ducking 规则：旁白期间 -12 dB，无人声时 -8 dB

### 4.8 时长与结构

| 视频长度 | 结构 |
|---|---|
| **15-30s（短视频）** | 1 个产品功能演示（钩子 + 演示 + CTA）|
| **1-3min（中视频）** | 多个功能演示（每个 20-30s）+ 数据展示 |
| **3min+（深度视频）** | 含技术实现细节（git log / 代码块）|

### 4.9 典型场景示例

- "七练的 3 秒记录是怎么实现的"
- "AI 怎么解析你的训练数据？"
- "周报功能演示：自动给你训练结论"
- "我从手动记 Excel 到做出七练的开发过程（git log）"
- "对比 Keep：七练的差异化在哪？"

## 5. 三类视频的对比速查表

| 维度 | A. 个人人设 | B. 健身知识 | C. 七练介绍 |
|---|---|---|---|
| **主体** | 人脸/口播 | 动作演示视频 | 弹出内容卡片 |
| **辅助** | 视频片段 | 关键参数 overlay | 健身视频背景 |
| **画面占比** | 人脸 60-70% | 动作 70%+ | 卡片 60% + 背景全画面 |
| **典型时长** | 15s-3min | 15s-3min | 15s-3min |
| **BGM** | Quiet Think | Power Build | Cyber Pulse |
| **字幕密度** | 中（同步口播）| 中低（重点词）| 高（解释卡片）|
| **人声角色** | 主体叙事 | 教学解释 | 引导观众看卡片 |
| **钩子类型** | 反常识/痛点共鸣/身份代入 | 痛点/数字冲击/悬念 | 反常识/数字冲击/悬念 |
| **CTA 强度** | 软 CTA | 弱 CTA（甚至无）| 强 CTA（产品引导）|

## 6. 与其他 rules 的联动

### 6.1 与 `copy.md` 协同

- 选题时先确定类型（A/B/C）
- 钩子类型由类型决定（见上表）
- 口播风格由类型决定：
  - A：讲故事 / 反思（第一人称为主）
  - B：教学 / 解释（带数据）
  - C：演示 / 引导（带截图引用）

### 6.2 与 `storyboard.md` 协同

每类视频的"分镜结构模板"参考对应章节：

| 类型 | 模板章节 |
|---|---|
| A. 个人人设 | 2.4 节"辅助视频片段切入口播" |
| B. 健身知识 | 3.3-3.5 节"动作演示 + 参数 overlay + 分屏" |
| C. 七练介绍 | 4.4-4.6 节"弹出内容类型 + 切换逻辑" |

### 6.3 与 `assets.md` 协同

每类视频的"素材优先级"：

| 类型 | 优先级 1（必备）| 优先级 2（增强）| 优先级 3（可选）|
|---|---|---|---|
| A | 口播视频 | 切入口播的动作片段 | 关键数据装饰图 |
| B | 训练动作视频（多角度）| 关键参数图 | 数据可视化 |
| C | 弹出内容（截图/代码/git log）| 健身背景视频 | 抽象动画背景 |

### 6.4 与 `bgm.md` 协同

| 类型 | 默认 BGM | 备选 |
|---|---|---|
| A | Quiet Think | Cyber Pulse（如果场景偏"理性产品")|
| B | Power Build | Hop Pulse（如果偏"步骤教程")|
| C | Cyber Pulse | Power Build（如果强调产品力量感）|

### 6.5 与 `subtitle.md` 协同

- A 类型：标准字幕（每条 ≤24 字、≤4s）
- B 类型：重点词高亮更密集（数字、动作名）
- C 类型：字幕密度可以略高（解释卡片内容时多打一些）

### 6.6 与 `script.md` 协同

- 配色 + 安全区 + 科技感/力量感 通用
- 不同类型的主体区域布局不同：
  - A：人脸居中，文字 / 数据角落
  - B：动作视频主体，参数 overlay 角落
  - C：弹出卡片居中，背景全画面

## 7. 混合类型

### 7.1 允许混合

实际视频**可能混用多个类型**：

- A → B：口播讲个人故事 → 切到动作演示
- B → C：动作演示 → 切到七练产品介绍
- A → C：口播讲理念 → 弹出七练功能演示

### 7.2 混合规则

- 主体类型 = 视频时长占比最大的类型
- 例：30s 中 20s 是动作演示、10s 是七练介绍 → 主体是 B
- 切换类型时**必须用转场**（≥0.3s，参考 `script.md` 第 8 节）
- 混合视频的 assets.md 要**两节都列**（"主体类型素材" + "辅助类型素材"）

## 8. 速查清单（只列**本文件专属**项）

> **跨文件去重原则**：通用检查见 [checklist.md](remotion/rules/checklist.md)；本节只列**视频类型选择**的专属项。

**确定类型**

- [ ] 用户说"做 X 视频"时，先判断属于 A/B/C 中的哪一类（见第 1 节）
- [ ] 主动告知用户分类结果，得到确认后再继续
- [ ] 混合类型时，明确主体类型 + 切换点（见第 7 节）

**按类型选 BGM**

- [ ] A → Quiet Think（默认，见第 2.6 节）
- [ ] B → Power Build（默认，见第 3.6 节）
- [ ] C → Cyber Pulse（默认，见第 4.7 节）

**按类型设计分镜**

- [ ] A：人脸 60-70% + 视频片段切入 + 角落数据 overlay（见第 2 节）
- [ ] B：动作视频 70%+ + 关键参数 overlay（动作名/重量/次数/RPE，见第 3 节）
- [ ] C：背景健身视频（弱化）+ 弹出内容卡片（中心 60%，见第 4 节）

**按类型收集素材**

- [ ] A：口播视频是必须的（用户自拍，见第 6.3 节）
- [ ] B：训练动作视频是必须的（用户自拍，多角度）
- [ ] C：弹出内容是必须的（截图/代码/git log/操作视频）+ 背景健身视频

**避免错误**

- [ ] A 类型不要把"产品功能截图"当主体（人脸才是）
- [ ] B 类型不要放太多文字（动作演示为主）
- [ ] C 类型不要用纯色背景（必须有弱化处理的健身视频背景，见第 4.3 节）
- [ ] 三类都不要违反 [script.md](remotion/rules/script.md) 第 2 节（无大面积纯色块遮挡）
