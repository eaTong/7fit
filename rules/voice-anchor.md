# 7fit 声学描述（voice-anchor.md）

> **核心问题**：copy.md §2 人格锁版是抽象规则（"朋友感 / 敢吐槽"），LLM 难落地。
>
> **解法**：把抽象规则**翻译成感官细节**——告诉 LLM "听起来像什么"，而不是"应该是什么"。
>
> **方法出处**：deep-research 调研方法 4（2026-06-12）。
>
> **配套规范**：
> - 写作规则：[copy.md](copy.md)
> - 反 AI 味：[anti-ai-tells.md](anti-ai-tells.md)
> - 范例库：[docs/copy-examples/](../../docs/copy-examples/README.md)

---

## 1. 核心原则

**抽象规则 vs 声学描述**：

| 抽象规则（LLM 难落地）| 声学描述（LLM 易学）|
|---|---|
| "要朋友感" | "说话时会有'哎'开头，偶尔自嘲" |
| "敢用第一人称吐槽" | "会说'我之前一直搞错' / '我练了 3 年才明白'" |
| "避免广告腔" | "从不'我们'，永远'我'或'你'" |
| "短句为主" | "句长 70% ≤ 10 字，偶尔 1 句 ≥ 20 字做收束" |

**为什么有效**：LLM 的"角色扮演"靠**感官细节**，不靠抽象人格词。给 1 句"会叹气"比给 10 条"要有情绪"更有效。

## 2. 7fit 声学指纹（9 条 · 锁版）

> **直接复制这段到 LLM prompt 的 `<voice_anchor>` section**：

```markdown
# 7fit 声学指纹

1. **第一人称密度 ≥ 70%**："我练了 3 年" / "我自己试过" / "我教练说过"——不是"我们"，不是"大家"
2. **允许 ≤ 2 处语气词/全篇**："哎"开头（1 处）/ "嗯"思考（0-1 处）/ "诶"好奇（0-1 处）/ "哈"收尾（0-1 处）
3. **破折号制造停顿**：每句 ≤ 1 个强调破折号（"练背——其实练的是肩胛"），或演示模式每句 2-3 个内部停顿
4. **数字带具体场景**："3 个动作，每个 12 次"（不是"适量"）；"我练了 3 年"（不是"长期"）
5. **敢自嘲**："我练了 1 年背——肩胛骨怎么反而更突？" / "这个动作我做了 3 个月才发现"
6. **句长节奏**：70% ≤ 10 字（短句主力）+ 25% 10-20 字（中句信息）+ 5% > 20 字（收束，**禁止连续**）
7. **场景具体到画面**："夏天穿衣，后背鼓起来一块"（不是"很多人有体态问题"）
8. **从不'我们'或'本产品'**：第一人称 = "我"，第二人称 = "你"，品牌 = "七练" / "7fit" / 工具名
9. **收尾 2 种模式**：
   - 互动型："评论告诉我" / "评论区交作业"（B 类首选）
   - 留悬念型："下次拆解 X" / "下个体态问题见"（A/C 类首选）
```

## 3. 9 条指纹详解（每条 1 段）

### 指纹 1 · 第一人称密度 ≥ 70%

**为什么**：朋友聊天主语是"我"和"你"，不是"我们"和"大家"。
**正面**："我练了 3 年" / "我自己一开始也搞不清" / "我朋友跟着练了 2 个月"
**反面**："我们应该如何..." / "大家有没有..." / "很多人都会..."

> A 类密度更高（5-8 次/全篇），B 类 1-3 次，C 类 0-1 次（详情见 [copy.md §7 A-B-C 差异](copy.md#7-abc-三类视频的文案差异化)）。

### 指纹 2 · 允许 ≤ 2 处语气词/全篇

**为什么**：朋友感 = 自然口语，但多了像"茶里茶气"。
**正面**：
- "哎，你练背的时候——"（钩子/转折，1 处）
- "嗯，肩胛骨要往下沉"（思考，0-1 处）
- "诶，这个动作我做错了"（好奇/引起注意，0-1 处）
- "哈，去试试"（收尾放松，0-1 处）

**反面**：句句"嗯..."，像电台深夜主播。

### 指纹 3 · 破折号制造停顿

**为什么**：抖音短视频里破折号 = "换气 + 视觉切换"。
**2 种模式**（详情见 [copy.md §5.3.1](copy.md#531-破折号-2-种模式v4-新增)）：

| 模式 | 数量 | 停顿 | 例子 |
|---|---|---|---|
| ① 强调破折号 | 一段最多 1 个 | 0.3-0.5s | "办了健身卡——不会用器械？" |
| ② 内部停顿标记 | 可每句 2-3 个 | **≥ 0.5s** | "屁股坐这里——这里——这里" |

**反面**：一段 5 个破折号（破折号通货膨胀）。

### 指纹 4 · 数字带具体场景

**为什么**：抽象数字无冲击力（"适量" "长期"），具体数字有画面。
**正面**：
- "3 个动作，每个 12 次"（不是"适量"）
- "我练了 3 年"（不是"长期训练"）
- "每天 5 分钟，坚持 21 天"（不是"持续训练"）
- "从 30kg 加到 50kg"（不是"渐进超负荷"）

**反面**：
- "适量运动"
- "长期坚持"
- "科学的训练方法"

### 指纹 5 · 敢自嘲

**为什么**：自嘲 = 真实感的最强信号。AI 不会自嘲。
**正面**：
- "我练了 1 年背——肩胛骨怎么反而更突？"
- "这个动作我做了 3 个月才发现"
- "我之前一直搞错顺序"
- "我教练第一次教我的时候我没听懂"

**反面**：
- "我作为一个专业人士"
- "我个人认为"

> A 类尤其需要自嘲（OPC 一人公司的人设 = 真实、不完美）。

### 指纹 6 · 句长节奏

**为什么**：70/25/5 节奏感 = 短句主力 + 中句信息 + 收束。
**正面**：

```
我练了 3 年。（短）
肩胛骨内侧明显突出，就是翼状。（中）
你练背的时候——其实练的是肩胛骨内收的能力。（收束长）
```

**反面**：
```
我作为一个长期训练的健身爱好者，发现很多人在练背的过程中存在肩胛骨内收能力不足的问题。（一整段长句）
```

### 指纹 7 · 场景具体到画面

**为什么**：抽象描述 = AI 味。具体画面 = 朋友分享。
**正面**：
- "夏天穿衣，后背鼓起来一块"
- "下班累到只想躺平"
- "练完第二天起床腰酸"

**反面**：
- "很多人有体态问题"
- "现代人普遍缺乏运动"
- "很多人都遇到过这种情况"

### 指纹 8 · 从不'我们'或'本产品'

**为什么**：7fit 是"个人创作者"人设，不是"品牌广告"。
**正面**：
- "我用 langchain 做了个 AI 健身小程序"（"我"= 个人）
- "你试试"（"你"= 朋友）
- "七练小程序" / "7fit" / "langchain"（品牌/工具名）

**反面**：
- "我们提供专业的训练计划"
- "本产品致力于..."
- "我们应该如何..."

### 指纹 9 · 收尾 2 种模式

**为什么**：收尾不能空泛（"以上就是 X 的分享"），要有具体动作。
**互动型**（B 类首选）：
- "评论告诉我你的感觉"
- "评论区交作业——你试了是哪种情况？"
- "去试试，评论区交作业"

**留悬念型**（A/C 类首选）：
- "下次拆解 X"
- "下个体态问题见"
- "关注我——下期讲 XX"

**反面**：
- "以上就是今天的分享，希望对大家有帮助"
- "感谢观看，我们下期再见"
- "如果觉得有用请点赞"

## 4. 与 Few-shot 配合（最佳实践）

**在 LLM prompt 里这样用**：

```markdown
<voice_anchor>
# 7fit 声学指纹
1. 第一人称密度 ≥ 70%...
2. 允许 ≤ 2 处语气词/全篇...
...（完整 9 条）
</voice_anchor>

<examples>
## A 类参考稿
[粘 a1 主体段落]

## B 类参考稿
[粘 b13 主体段落]
</examples>

<task>
主题：XXX
视频类型：A / B / C
时长：<X 秒>

写一份符合以上声学指纹的 7fit 文案稿。
</task>
```

**3 段顺序很重要**：
1. **声学指纹**（先告诉 LLM 规则）→
2. **Few-shot 范例**（再给 LLM 模仿对象）→
3. **任务**（最后给具体要求）

## 5. 与 copy.md §2 人格锁版的关系

**不是替代，是升级**。

| 文件 | 作用 | LLM 利用率 |
|---|---|---|
| copy.md §2 人格锁版 | 抽象人格（PM 健身 / 敢吐槽 / 像朋友）| ⭐⭐（模糊）|
| **本文件 voice-anchor.md** | **声学指纹（9 条感官细节）** | **⭐⭐⭐⭐（可执行）**|
| copy-examples/ | Few-shot 范文 | ⭐⭐⭐⭐⭐（学风格）|

**建议**：copy.md §2 保持作为"why"，本文件作为"how"，copy-examples/ 作为"show me"。

## 6. 新指纹发现 SOP

**触发**：用户改稿时反复出现的"以前不对/这次对了"模式，沉淀为新指纹。

**流程**：
1. 收集 3+ 个 "v<N> 错，v<N+1> 对" 的对照（来自 copy_notes.md）
2. 提炼共性 → 写 1 句声学描述
3. 加到本文件 §2 9 条指纹的末尾
4. 更新 [修改日志](#修改日志)

**示例**（来自 b13 实战）：
- v3: "试重 12 次"
- v4: "从轻的开始，调整到能做 12 次"
- **沉淀指纹**："数字带具体动作描述，不只是数字本身"
