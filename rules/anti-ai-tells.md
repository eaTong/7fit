# 反 AI 味清单（anti-ai-tells.md）

> **核心问题**：LLM 生成文案默认会调用"训练数据里出现频率最高的表达"——这些表达往往**很顺、很对、但很假**。
>
> **解决思路**：在 prompt 末尾**显式禁止**这些表达，强制 LLM 走"少用但真实"的小众分布。
>
> **方法出处**：deep-research 调研方法 2（2026-06-12）。
>
> **配套规范**：
> - 写作规则：[copy.md](copy.md)
> - 声学描述：[voice-anchor.md](voice-anchor.md)
> - 范例库：[docs/copy-examples/](../../docs/copy-examples/README.md)

## 2. 禁用句式（8 条）

> **判断标准**：看到这个句式，90% 是 AI 写的。

| # | 禁用句式 | 为什么 AI 味 | 替代方案 |
|---|---|---|---|
| 2.1 | "首先 / 其次 / 此外 / 总之 / 综上所述" | 议论文标志，聊天不用 | 用破折号或直接换行 |
| 2.2 | "X 不仅...而且..." | 对称句式，训练数据高频 | 拆成 2 短句："X 是 Y。也 Z。" |
| 2.3 | "作为一个..." / "在当今..." / "随着 X 的发展" | 教科书开场 | 直接进内容 |
| 2.4 | "让我们..." | 命令受众，反口语 | "我们去..." / "你试试..." |
| 2.5 | "总而言之 / 一言以蔽之 / 归根结底" | 总结癖 | 用"反正" / "说白了" / 不总结 |
| 2.6 | "值得注意的是 / 需要指出的是 / 不难发现" | 学术腔 | 删掉，直接说结论 |
| 2.7 | "毋庸置疑 / 显而易见 / 众所周知" | 故作权威 | 删掉 |
| 2.8 | "我在这里想说..." / "我想分享的是..." | 凑字 | 直接说观点 |

## 3. 禁用词（12 条）

> **判断标准**：在 7fit 文案里出现 ≥ 1 次就违和。

### 3.1 营销废话（6 条）

| # | 禁用词 | 为什么 AI 味 | 替代方案 |
|---|---|---|---|
| 3.1.1 | **极致** | 烂大街 + 抖音违禁 | 删掉，或用具体词（"我练了 3 年"）|
| 3.1.2 | **颠覆 / 重塑 / 革新** | 营销词 + 抖音违禁 | 用事实："以前要 3 步，现在 1 步" |
| 3.1.3 | **赋能 / 抓手 / 闭环** | 互联网黑话 | 删掉，PM 朋友聊天不会说 |
| 3.1.4 | **深度 / 高度 / 广度** | 抽象大词 | 删掉 |
| 3.1.5 | **全方位 / 立体化 / 矩阵式** | 假大空 | 删掉 |
| 3.1.6 | **解决方案 / 工具 / 平台** | 自家产品不当产品词用 | 改成具体名（"langchain" / "7fit 小程序"）|

### 3.2 学术腔（3 条）

| # | 禁用词 | 为什么 AI 味 | 替代方案 |
|---|---|---|---|
| 3.2.1 | **研究表明 / 数据显示 / 专家建议** | 没人看的伪引用 | 用个人经验："我自己试了 6 个月" |
| 3.2.2 | **一定程度上 / 在某种程度上** | 模糊 | 给具体数字或删 |
| 3.2.3 | **诸如 / 诸如此类 / 等等** | 凑字 | 列出全部或用"几个" |

### 3.3 励志空话（3 条）

| # | 禁用词 | 为什么 AI 味 | 替代方案 |
|---|---|---|---|
| 3.3.1 | **坚持就是胜利 / 努力就会有回报** | 陈词滥调 | 用具体动作："每天 5 分钟" |
| 3.3.2 | **相信自己 / 你可以的** | 说教 | 删掉，让用户自己判断 |
| 3.3.3 | **梦想 / 热爱 / 初心** | 鸡汤 | 删掉 |

## 4. 禁用结构（6 条）

| # | 禁用结构 | 为什么 AI 味 | 替代方案 |
|---|---|---|---|
| 4.1 | **总-分-总 议论文** | 八股文，PM 朋友不会 | 短句 + 列举 + 收束 |
| 4.2 | **3 段以上用相同句式开头** | 节奏感差 | 每段开头换（陈述/反问/动作/数字）|
| 4.3 | **段落以"因此 / 所以"开头** | 过度论证 | 删掉连接词，直接说结论 |
| 4.4 | **每段都是问句开头** | 抖音被骂"AI 装互动" | 问句 ≤ 1 个/全篇 |
| 4.5 | **完美对称的 2 段 / 3 段** | 看起来假 | 打破对称，加 1 句口语转折 |
| 4.6 | **金句堆砌（连续 3 句无连接）** | 像 PPT 排比 | 用破折号或叙述句连接 |

### 5.1 开头（3 条）

| # | 禁用 | 为什么 AI 味 | 替代方案 |

| # | 禁用 | 为什么 AI 味 | 替代方案 |
|---|---|---|---|
| 5.2.1 | "以上就是 X 的分享，希望对大家有帮助" | 教学视频感 | 用具体 CTA（"评论告诉我" / "去试试"）|
| 5.2.2 | "感谢观看，我们下期再见" | 老派电视感 | 用留悬念（"下个体态问题见"）|

## 6. 自检脚本（Python）

> **触发**：copy.md §10.1 改稿 SOP 第 4 步"跑 5 维评分卡"后，跑这个脚本。

```python
#!/usr/bin/env python3
"""
anti_ai_tells_check.py · 7fit 文案反 AI 味自检
用法：python tools/anti_ai_tells_check.py <copy.md>
输出：每条命中的规则 + 行号 + 原文 + 严重度
"""

import re
import sys
from pathlib import Path

# 规则库（与 anti-ai-tells.md 同步）
PATTERNS = [
    # §2 句式
    (r'首先|其次|此外|总之|综上所述', '§2.1 议论文标志', 'high'),
    (r'不仅.{1,20}而且', '§2.2 对称句式', 'high'),
    (r'作为一个|在当今|随着.{1,10}的发展', '§2.3 教科书开场', 'high'),
    (r'让我们', '§2.4 命令受众', 'medium'),
    (r'总而言之|一言以蔽之|归根结底', '§2.5 总结癖', 'medium'),
    (r'值得注意的是|需要指出的是|不难发现', '§2.6 学术腔', 'medium'),
    (r'毋庸置疑|显而易见|众所周知', '§2.7 故作权威', 'medium'),
    (r'我想说的是|我想分享的是', '§2.8 凑字', 'low'),
    # §3 词
    (r'极致|颠覆|重塑|革新', '§3.1.1-2 营销废话', 'high'),
    (r'赋能|抓手|闭环', '§3.1.3 互联网黑话', 'high'),
    (r'全方位|立体化|矩阵式', '§3.1.5 假大空', 'medium'),
    (r'研究表明|数据显示|专家建议', '§3.2.1 伪引用', 'high'),
    (r'一定程度上|在某种程度上', '§3.2.2 模糊', 'medium'),
    (r'坚持就是胜利|努力就会有回报', '§3.3.1 鸡汤', 'high'),
    # §4 结构（用 grep 难检测，靠 LLM 评审）
    # §5 开头/收尾
    (r'^大家好[，,]', '§5.1.1 寒暄式开场', 'high'),
    (r'^今天我们来', '§5.1.2 客套', 'medium'),
    (r'以上就是.{1,20}的分享', '§5.2.1 教学视频感', 'high'),
    (r'感谢观看', '§5.2.2 老派电视感', 'high'),
]

def check(file_path: Path) -> list[dict]:
    findings = []
    text = file_path.read_text(encoding='utf-8')
    lines = text.split('\n')
    for line_num, line in enumerate(lines, 1):
        # 跳过注释/标题/元信息
        if line.startswith('#') or line.startswith('>') or line.startswith('---'):
            continue
        for pattern, rule, severity in PATTERNS:
            matches = re.findall(pattern, line)
            if matches:
                findings.append({
                    'line': line_num,
                    'rule': rule,
                    'severity': severity,
                    'match': matches[0] if isinstance(matches[0], str) else matches[0][0],
                    'text': line.strip()[:100]
                })
    return findings

if __name__ == '__main__':
    if len(sys.argv) < 2:
        print('Usage: anti_ai_tells_check.py <copy.md>')
        sys.exit(1)
    file_path = Path(sys.argv[1])
    findings = check(file_path)
    if not findings:
        print(f'✅ {file_path.name}: 0 命中')
        sys.exit(0)
    # 按严重度排序
    severity_order = {'high': 0, 'medium': 1, 'low': 2}
    findings.sort(key=lambda f: (severity_order[f['severity']], f['line']))
    print(f'⚠️  {file_path.name}: {len(findings)} 命中\n')
    for f in findings:
        print(f"  L{f['line']} [{f['severity'].upper()}] {f['rule']}")
        print(f"    命中: 「{f['match']}」")
        print(f"    原文: {f['text']}")
        print()
    # exit code：high 命中 → 1
    high_count = sum(1 for f in findings if f['severity'] == 'high')
    sys.exit(1 if high_count > 0 else 0)
```

**集成到改稿 SOP**：

```bash
# 改稿后跑自检
python tools/anti_ai_tells_check.py docs/copy/<主题>.md
echo "Exit code: $?"  # 0 = 干净, 1 = 有 high 命中必须改
```

## 7. Few-shot 原理（为什么"反 AI 味清单"还不够）

> **核心洞察**：光禁止 AI 常用表达，只能让 LLM "不说 AI 话"，**不能让它说"你的话"**。
>
> **真正有效**的是把"少用但真实"的小众分布显式喂给 LLM：

| 层级 | 方法 | 效果 |
|---|---|---|
| L1 | 只给规则（copy.md）| ⭐⭐（避免最差）|
| L2 | L1 + 反 AI 味清单（本文件）| ⭐⭐⭐（不说 AI 话）|
| L3 | L2 + voice-anchor.md（声学描述）| ⭐⭐⭐⭐（说"像你"的话）|
| **L4** | **L3 + copy-examples/（Few-shot 范例）**| **⭐⭐⭐⭐⭐（学你的风格）**|

**所以本清单必须与以下 3 个文件配套使用**：
- [voice-anchor.md](voice-anchor.md) —— 告诉 LLM "听起来像什么"
- [docs/copy-examples/](../../docs/copy-examples/README.md) —— 给 LLM 看 3 篇范文
- [copy.md §11 模板](copy.md#11-文案稿结构模板) —— 强制具体性三件套

## 8. 改稿循环（与 LLM 自评配合）

```
1. LLM 生成初稿
   ↓
2. 跑本自检脚本（§6）
   ↓ high 命中
3. 提示 LLM："命中 §2.1「其次」，请改用口语化连接"
   ↓
4. LLM 改稿
   ↓
5. 再跑自检 → 干净 → 进入用户审阅
```
