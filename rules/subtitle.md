# 字幕生成规范（subtitle.md）

> **Phase 3 核心产物**：**从 copy.md 文案直接生成** `subtitles.json`（**无需 STT**）—— 录旁白只用于后续对齐音频用。
>
> **生成工具**：[`tools/gen-subtitles.js`](../../tools/gen-subtitles.js)（一次性脚本，3 大能力：解析 copy / 计算时长 / 自动标 highlight）
>
> **必须遵循**：[timing-sync.md](timing-sync.md)（每条字幕时长 = 字数 / 中速 + 破折号停顿）+ [copy.md §15 下游接口](copy.md#15-下游接口说明)（重点词→highlight 强制）
>
> **字幕哲学**：**字幕是"无声讲解员"**——让用户关掉声音也能看懂 80% 的内容。3 要素：**清晰**（≥ 28px）+ **重点**（highlight 强制）+ **节奏**（时长 = 字数 / 3.4）。

---

## 1 · 工作流

```
copy.md 文案定稿（v6+）
  ↓
【自动】tools/gen-subtitles.js <主题>
  ├─ 解析 ## 钩子 / ## 主体（含 ### 段 N） / ## 收尾
  ├─ 每 section 第一个 > 块 = 实际脚本（其余 = metadata 跳过）
  ├─ 计算每句时长 = speech_chars / speed + dash_count * 0.5
  ├─ 段间停顿 0.7s（仅 section 边界）
  ├─ 自动检测 highlight（PROJECT_HIGHLIGHTS 候选 + 去重叠）
  └─ 字段校验：每条 ≤ 4s + ≤ 24 字 + ≤ 2 highlights
  ↓
remotion/src/scenes/<主题>/subtitles.json
  ↓
【可选】录旁白 → resources/audios/<主题>.m4a（仅用于视频合成，**不影响字幕 timing**）
  ↓
Scene 组件加载 + spring 弹跳入场
  ↓
5 维评分卡自评（≥ 18 分才能进用户审阅）
```

> **为什么不需要 STT**：
> - copy.md 的标点（句号、破折号）就是**确定的**句界和停顿点
> - 计算时长 = `字数 / 3.4 + 破折号 × 0.5`（deterministic）
> - 录旁白只是**音频轨道**，与字幕 timing **解耦**——音频时长可以略超/略短字幕，Remotion 引擎独立 seek
> - 之前用 Whisper 实际识别（基于录音）反而引入"用户念错的字 → STT 错字"的污染链，且不匹配 timeline 计划

## 2 · JSON 格式

### 2.1 数据结构

```json
{
  "id": "winged_scapula_b3",
  "language": "zh-CN",
  "duration": 65.4,
  "track_index": 0,
  "subtitles": [
    {
      "id": "sub_001",
      "start": 0.0,
      "end": 3.0,
      "text": "你以为靠墙就能改翼状肩？",
      "segments": [
        { "text": "你以为靠墙就能改翼状肩？", "highlight": true }
      ]
    },
    {
      "id": "sub_002",
      "start": 3.5,
      "end": 4.97,
      "text": "练了 3 个月",
      "segments": [
        { "text": "练了 3 个月", "highlight": false }
      ]
    },
    {
      "id": "sub_003",
      "start": 5.0,
      "end": 7.06,
      "text": "肩胛还是凸出来",
      "segments": [
        { "text": "肩胛还是凸出来", "highlight": true }
      ]
    }
  ]
}
```

> **说明**：原 sub_002 "练了 3 个月，肩胛还是凸出来"（12 字 / 6s = 2.0 字/秒）违反 [timing-sync.md §6](timing-sync.md#6--单条字幕时长上限单源真相--从-subtitlemd-61-迁来) 的"单条 ≤ 4s + 24 字 + 中速 3.4 字/秒"。已拆为 sub_002（5 字 / 1.47s）+ sub_003（7 字 / 2.06s）两条独立字幕。

### 2.2 字段说明

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `id` | string | ✅ | 字幕条 ID（全局唯一，推荐 `sub_<3 位数字>`）|
| `start` | number | ✅ | 开始时间（秒）|
| `end` | number | ✅ | 结束时间（秒）|
| `text` | string | ✅ | 完整文本（用于 SEO / 复用）|
| `segments[]` | array | ✅ | 分段（用于 highlight）|
| `segments[].text` | string | ✅ | 分段文本 |
| `segments[].highlight` | bool | ✅ | 是否高亮（重点内容）|

### 2.3 字段值约束

| 字段 | 约束 |
|---|---|
| `id` | 全局唯一，`sub_001` 起步 |
| `start` / `end` | 来自 mmx 转写，**手动校验 ≤ 4s + ≤ 24 字** |
| `text` | 无句末标点（参考 §7）|
| `segments[].highlight` | 1 条字幕 ≤ 2 个 highlight segment |

## 3 · 字幕样式硬约束

| 维度 | 规格 |
|---|---|
| **字号** | ≥ 28px（移动端可读） |
| **颜色** | 纯白 `#FFFFFF` |
| **字重** | 默认 500 / highlight 700 |
| **highlight 字号** | 1.15× 默认 |
| **highlight 颜色** | 橙色 `#FF4500` |
| **阴影** | `text-shadow: 0 2px 8px rgba(0,0,0,0.8)` |
| **位置** | 底部居中（≥ 80px bottom 安全区） |
| **最大行数** | 2 行 |
| **最大字数** | 单条 ≤ 24 字 |

### 3.1 主体区域让位

**配字幕时，主体区域不放长文字**——字幕占底部 80px+ 空间，主体画面要让位。

### 3.2 字幕容器规范

```html
<div class="subtitle" data-shot-id="self_test_1" data-start="3.5" data-duration="6.0">
  <span class="subtitle__text">背对镜子站好，肩膀放松手垂下来</span>
  <span class="subtitle__highlight">肩胛骨内侧明显突出</span>
</div>
```

```css
.subtitle {
  position: absolute;
  bottom: 80px;
  left: 50%;
  transform: translateX(-50%);
  font-size: 32px;
  font-weight: 500;
  color: #FFFFFF;
  text-shadow: 0 2px 8px rgba(0, 0, 0, 0.8);
  text-align: center;
  max-width: 90%;
  line-height: 1.3;
}
.subtitle__highlight {
  font-size: 36.8px;  /* 1.15 × 32 */
  font-weight: 700;
  color: #FF4500;
}
```

## 4 · 重点 segment 标记规则

打 `highlight: true` 的内容：

| 类型 | 示例 | 频次（全篇）|
|---|---|---|
| 数字 | "3 步" / "10 分钟" / "170 字" | 5-10 个 |
| 动作 | "靠墙天使" / "俯身 Y-T-W" / "壁虎推墙" | 3-6 个 |
| 品牌句 | "用产品思维去健身" | 0-1 个 |
| CTA | "评论区交作业" / "去试试" | 1-2 个 |
| 反常识点 | "错了" / "不科学" / "基本没跑了" | 1-3 个 |
| 安全提示 | "请就医" / "别自己硬练" | 0-1 个 |

### 4.1 highlight 频次铁律

- **1 条字幕 ≤ 2 个 highlight segment**
- **全篇 highlight ≤ 30%**（太多→失去强调作用）

**反例**：
- ❌ 1 条字幕 3 个 highlight → 眼花
- ❌ 全篇 50% highlight → 处处是重点 = 没重点

### 4.2 highlight 强制清单（来自 [copy.md §15](copy.md#15-下游接口说明) 下游接口）

| 字段来源 | 高亮位置 |
|---|---|
| 文案稿标注的 `[xxx]` 重点词 | 必 highlight |
| 文案稿的次数/组数（"12 次 3 组"）| 必 highlight |
| 文案稿的 CTA 句 | 必 highlight |
| 文案稿的安全提示 | 必 highlight |

## 5 · 入场动效（Remotion spring + interpolate）

### 5.1 默认入场

```tsx
// spring 弹跳入场
const enterProgress = interpolate(localFrame, [0, 10], [0, 1], {
  extrapolateLeft: "clamp",
  extrapolateRight: "clamp",
  easing: Easing.out(Easing.cubic),
});
const y = interpolate(enterProgress, [0, 1], [30, 0]);
const opacity = enterProgress;
```

### 5.2 highlight segment 二次跳动

```tsx
// highlight 二次跳动（back.out spring）
const highlightProgress = interpolate(localFrame, [0, 7, 14], [0, 1, 0], {
  extrapolateLeft: "clamp",
  extrapolateRight: "clamp",
  easing: Easing.out(Easing.back(1.7)),
});
const scale = interpolate(highlightProgress, [0, 1], [1, 1.15]);
```

### 5.3 出场

```tsx
// 出场：向上滑出 + 淡出
const exitProgress = interpolate(localFrame, [durationFrames - 9, durationFrames], [1, 0], {
  extrapolateLeft: "clamp",
  extrapolateRight: "clamp",
  easing: Easing.in(Easing.cubic),
});
const y = interpolate(exitProgress, [0, 1], [0, -20]);
const opacity = exitProgress;
```

### 5.4 动效选型（按字幕类型）

| 字幕类型 | 入场动效 | 时长 |
|---|---|---|
| **普通字幕** | y: 30→0 + opacity: 0→1 | 0.35s |
| **highlight** | back.out(1.7) 二次弹跳 | 0.25s |
| **数字滚动** | snap + scale: 0.5→1.15 | 0.4s |
| **CTA 大字幕** | y: 50→0 + scale: 0.8→1 | 0.5s |
| **警示字幕** | x: ±20→0 + opacity: 0→1 | 0.3s |

## 6 · 时长校验公式

> **本节是 subtitle 视角的"如何用"，单条硬上限在 [timing-sync.md §6](timing-sync.md#6--单条字幕时长上限单源真相--从-subtitlemd-61-迁来)（单源真相）。

### 6.1 公式校验

```js
// 主体区段：每条字幕的 (end - start) ≈ text.length / 3.4
// 例：6 字文本 → 6 / 3.4 ≈ 1.76s → 实际 1.7-1.8s 合理
// 例：24 字文本 → 24 / 3.4 ≈ 7.06s → 实际 7.0-7.1s 合理
// 允许 ± 0.3s 误差
```

### 6.2 单条时长上限（见 timing-sync.md §6.1）

- **单条 ≤ 4 秒**（用户阅读时间）
- **单条 ≤ 24 字**（视觉密度）

> **超出**：必须拆成多条（详 §9.1 长句拆条策略）。

## 7 · 违禁词 / 标点规范

- **不带句末标点**（字幕显示"练了 3 个月"而不是"练了 3 个月。"）
- **允许的语气词**：嗯 / 诶 / 哎 / 哈
- **禁用**：感叹号叠加（"！！"）、emoji、乱码
- **禁用**：违禁词（参考 [copy.md §12 违禁词清单](copy.md#12-违禁词清单13-类高优处理)）

### 7.1 标点使用细则

| 标点 | 用法 |
|---|---|
| **逗号** | 允许，用于长句内部停顿 |
| **句号** | 禁用（字幕不带句末）|
| **问号** | 钩子/CTA 专用 |
| **破折号** | 允许（"——"），1 段最多 1 个 |
| **感叹号** | 禁用（显得不自然）|
| **省略号** | 禁用（"……" 显懒）|

## 8 · 拆条与合并策略

> **核心问题**：mmx 转写出来的字幕常违反单条 ≤ 24 字 + ≤ 4s 的铁律。本节给出拆 / 合的判断标准。

### 8.1 长句拆条策略

**触发**：单条 > 24 字 **或** > 4s。

**拆分原则**：
- 按语义断句（不强行按字数拆）
- 关键概念词保留在同一段（避免歧义）
- 拆完后每条 ≤ 24 字

**示例**：

| ❌ 拆前（36 字 / 10s） | ✅ 拆后（3 条） |
|---|---|
| "背对镜子站好，肩膀放松手垂下来，肩胛骨内侧明显突出，就是翼状" | 1. "背对镜子站好，肩膀放松手垂下来"（14 字 / 4.1s）<br>2. "肩胛骨内侧明显突出"（9 字 / 2.6s）<br>3. "就是翼状"（4 字 / 1.2s）|

### 8.2 短句合并策略

**触发**：相邻 2 条字幕都 < 1.5s **且** 语义连贯。

**合并原则**：
- 合并后 ≤ 24 字
- 保留原 highlight 标记
- 段间停顿不能合并（无 pause_breath 时）

**示例**：

| ❌ 拆前（2 条，2.3s + 1.4s）| ✅ 合后（1 条，3.7s）|
|---|---|
| 1. "壁虎推墙"（1.3s）<br>2. "前臂贴墙"（1.4s）| "壁虎推墙，前臂贴墙"（9 字 / 2.6s）|

### 8.3 拆 / 合决策表

| 情况 | 处理 |
|---|---|
| 单条 > 24 字 | **必拆** |
| 单条 > 4s | **必拆** |
| 2 条都 < 1.5s 且语义连贯 | **可合** |
| 含 CTA / 数字 / 动作名 | **不合**（保留独立时序）|
| 含安全提示 | **不合**（独立时序，确保观众看到）|

## 9 · 字幕样式库（5 类）

### 9.1 钩子字幕（0-3s）

| 维度 | 规格 |
|---|---|
| 字号 | 48-64px（普通段落的 1.5-2×）|
| 字重 | 700（bold）|
| 颜色 | 纯白 + 关键 highlight 橙色 |
| 位置 | 中央（不底部，钩子要抢眼）|
| 动效 | 弹跳入场（back.out 1.4, 0.5s）|

**反例**：
- ❌ 钩子用底部位置 → 不抢眼
- ❌ 钩子用 32px → 看不清

### 9.2 普通字幕

| 维度 | 规格 |
|---|---|
| 字号 | 32px（≥ 28px 硬约束）|
| 字重 | 500 |
| 颜色 | 纯白 |
| 位置 | 底部 80px |
| 动效 | y: 30→0, 0.35s |

### 9.3 highlight 字幕

| 维度 | 规格 |
|---|---|
| 字号 | 36.8px（1.15× 默认）|
| 字重 | 700 |
| 颜色 | 橙色 `#FF4500` |
| 位置 | 同普通字幕（在同一行内）|
| 动效 | 二次弹跳（yoyo + repeat: 1）|

### 9.4 数字字幕（"12 次 × 3 组"）

| 维度 | 规格 |
|---|---|
| 字号 | 48px（强调）|
| 字重 | 700 |
| 颜色 | 橙色 + 数字滚动 |
| 位置 | 右上角卡片（与文字字幕分离）|
| 动效 | snap + scale: 0.5→1.15 |

> 数字字幕常与右上角"数据卡"组件共用，**不**作为主字幕。

### 9.5 警示字幕（"请就医" / "别自己硬练"）

| 维度 | 规格 |
|---|---|
| 字号 | 28-32px |
| 字重 | 700 |
| 颜色 | 电红 `#DC143C` |
| 背景 | `rgba(220, 20, 60, 0.15)` |
| 位置 | 底部上方 200px（不与主字幕同高）|
| 动效 | x: ±20→0（左右滑入）|

## 10 · A/B/C 三类字幕差异化

| 维度 | A 人设 | B 知识 | C 产品 |
|---|---|---|---|
| **人称密度** | 高（"我/你"）| 中（"你/动作"）| 低（"功能"）|
| **highlight 频次** | 低（5-8 处/全篇）| 高（10-15 处/全篇）| 中（8-12 处/全篇）|
| **数字频次** | 低（"练了 3 年"）| 高（"3 步 + 4 动作 + 10 分钟"）| 中（"100 次训练"）|
| **警示频次** | 低 | **高**（健身赛道必加安全提示）| 低 |
| **CTA 形式** | 互动（"评论告诉我"）| 互动（"评论交作业"）| 留悬念（"下期讲" + 引流）|
| **典型字数** | 180-220 字 | 200-260 字 | 150-200 字 |
| **典型条数** | 25-30 条 | 30-40 条 | 20-30 条 |

### 10.1 A 类字幕风格

> 例："我练了 3 年，肩胛骨反而越练越突"
> - highlight: "3 年"、"越练越突"
> - 语气：第一人称吐槽

### 10.2 B 类字幕风格

> 例："背对镜子站好，肩胛骨内侧明显突出，就是翼状"
> - highlight: "背对镜子"、"肩胛骨内侧"、"翼状"
> - 语气：教学 + 动作指令

### 10.3 C 类字幕风格

> 例："我用 7fit 记了 100 次训练，3 个宝藏功能"
> - highlight: "7fit"、"100 次"、"3 个宝藏功能"
> - 语气：种草 + 数字冲击

## 11 · 5 维评分卡 + 评审 SOP

> **每维 ≥ 3 分才能进入用户审阅**，总分 ≥ 18/25。

### 11.1 评分卡

| 维度 | 1 分（差）| 3 分（中）| 5 分（优）| 本稿得分 |
|---|---|---|---|---|
| **时长合规** | > 30% 条违反 ≤ 4s | < 10% 违反 | 0 违反（自动校验过）| — |
| **字数合规** | > 30% 条违反 ≤ 24 字 | < 10% 违反 | 0 违反 | — |
| **highlight 合理** | 全篇 50% highlight | 全篇 20-30% | 全篇 20-25% + 关键 4 类齐全 | — |
| **样式规范** | 字号 < 28px / 句末带标点 | 全部 ≥ 28px + 无句末标点 | 全部规范 + 5 类样式选型对 | — |
| **动效同步** | 用 CSS transition / 漏动效 | 全用 interpolate + 0.3-0.4s | 全用 interpolate + 5 类选型对 + 节奏匹配段间 | — |

### 11.2 评审 SOP

```
1. 跑自动校验（[timing-sync.md §6.1](timing-sync.md)）
   - 单条 > 24 字 → 标红
   - 单条 > 4s → 标红
   ↓
2. 自评 5 维（≥ 18 分）
   ↓
3. Studio 加载全屏过 3 遍
   - 钩子段：3s 看清？
   - 自测段：动作与字幕同步？
   - 动作段：highlight 弹跳明显？
   - 收尾段：CTA 醒目？
   ↓
4. 用户审阅 → 通过 / 改稿
```

## 12 · 反模式

- ❌ 单条字幕 > 24 字（视觉过密）
- ❌ 单条字幕 > 4s（用户读完会走神）
- ❌ 字幕带句末标点
- ❌ 字号 < 28px（移动端不可读）
- ❌ 全部 highlight（高亮无意义）
- ❌ 用 CSS transition / animation 做入场（remotion 按帧渲染时不按帧推进）
- ❌ 不校验时长直接用 mmx 输出（节奏可能与 [timing-sync.md](timing-sync.md) 锚点不符）
- ❌ 录完旁白才发现节奏不对，回头改文案但不改字幕时间
- ❌ **1 条字幕 3 个 highlight segment**（眼花）
- ❌ **长句不拆直接用**（"36 字"违反 ≤ 24 字铁律）
- ❌ **警示字幕与主字幕同高**（互相遮挡）
- ❌ **钩子用底部位置**（不抢眼，3s 看不完）
- ❌ **不写 segments 数组，全 highlight=false**（等于没高亮）
- ❌ **跳过 5 维评分卡直接给用户**
- ❌ **违禁词进字幕**（[copy.md §12](copy.md#12-违禁词清单13-类高优处理)）
