# 拍摄清单（shoot-checklist.md）

> **Phase 4 配套产物**：用户自拍训练动作 / 口播视频前，**必须**先过本清单 7 项检查。
>
> **存放位置**：`remotion/src/scenes/<主题>/shoot-checklist.md`（每个 scene 自带一份）。
>
> **目的**：避免**拍完不能用**（光线差 / 机位错 / 时长短 / 动作不标准）造成的返工。
>
> **关联**：[assets.md §5](assets.md#5--缺失的视频素材用户自拍) + [video-types.md §关键素材](video-types.md#关键素材硬约束) + [rhythm.md §3.2 素材完成门控](rhythm.md)
>
> **拍摄哲学**：**宁可多拍 10 分钟，不要后期重拍 1 小时**。3 要素：**5 维规格全填**（机位/光线/时长/动作/其他）+ **多角度多版本**（后期选择多）+ **验收必走**（不达标准立即重拍）。

---

## 1 · 5 维拍摄规格（必填项）

每个素材文件拍摄前，**必须**明确以下 5 维：

| 维度 | 必填内容 | 例 |
|---|---|---|
| **机位** | 拍摄角度 + 距离 | "正面 1.5m / 侧面 1.2m / 俯视 45°" |
| **光线** | 自然光 / 灯光 / 时段 | "上午 10 点窗边自然光 / 主光在右 45°" |
| **时长** | 录制时长（秒）| "8s（包 1 次完整动作）" |
| **动作** | 完整动作描述 | "靠墙天使 1 次完整动作：双手侧平举贴墙向上滑到头顶再下放" |
| **其他** | 服装 / 背景 / 道具 | "深色紧身衣 / 黑背景 / 无道具" |

> **5 维全填才能开拍**。漏一项可能拍完不可用。

### 1.1 5 维扩展项（视场景加填）

| 维度 | 扩展内容 | 适用场景 |
|---|---|---|
| **机位** | 横屏/竖屏 / 锁定旋转 | 默认竖屏 |
| **光线** | 顺光/逆光/侧光 / 色温 | 主光在右 45° |
| **时长** | 包含完整动作次数 | 1 次完整动作 = 8s |
| **动作** | 起始姿势 / 终止姿势 / 节奏 | 起→过程→收 |
| **其他** | 同期声 / 静音 / 道具 / 服装色 | 默认静音（后期配旁白）|

## 2 · 拍摄前 7 项检查

```
[ ] 1. 5 维规格填齐（机位 / 光线 / 时长 / 动作 / 其他）
[ ] 2. 设备电量 ≥ 50%（iPhone 建议 ≥ 80%）
[ ] 3. 存储空间 ≥ 2GB（1080p 视频 1 分钟 ≈ 100MB）
[ ] 4. 横屏 / 竖屏确认（项目默认竖屏 1080×1920）
[ ] 5. 麦克风 / 收音确认（同期声场景必查；纯动作不用）
[ ] 6. 拍摄场地就位（背景干净 / 无杂物 / 光线稳定）
[ ] 7. 演员就位（动作熟练 / 服装对 / 表情自然）
```

> 7 项全部 ✅ 才能按录制键。

### 2.1 7 项检查详解

| # | 检查项 | 失败影响 | 修法 |
|---|---|---|---|
| 1 | 5 维规格填齐 | 拍完发现机位错 / 时长短 | 补全规格 |
| 2 | 设备电量 ≥ 50% | 录到一半关机 | 充电 / 换设备 |
| 3 | 存储空间 ≥ 2GB | 录到一半空间满 | 清存储 |
| 4 | 竖屏确认 | 拍成横屏 | 锁定旋转（控制中心）|
| 5 | 收音确认 | 同期声录不上 / 杂音大 | 换设备 / 后期去噪 |
| 6 | 场地就位 | 背景杂乱 / 光线变化 | 换场地 / 调时段 |
| 7 | 演员就位 | 动作不标准 / 表情僵硬 | 试做 2 次 |

## 3 · 拍摄时 4 条铁律

### 3.1 拍多角度 / 多版本

| 场景 | 拍几个版本 |
|---|---|
| **主动作** | 至少 3 个角度（正面 / 侧面 / 特写）|
| **自测** | 至少 1 个完整 + 1 个特写 |
| **数据 overlay** | 至少 1 个静态（30s+） |
| **收尾 / CTA** | 1 个就够（口播或动作定格）|

> **多版本 = 后期选择多**。宁可多拍 10 分钟，不要后期发现角度不够重拍 1 小时。

### 3.2 拍稳定 / 防抖

- **iPhone**：用三脚架或稳定器，**禁用**手持拍主动作
- **稳定时长**：每段 ≥ 5s（[storyboard.md §2](storyboard.md#2-时长硬约束) 视频类镜头硬约束）
- **开头 / 结尾留 1s 缓冲**（后期切黑边 + 转场）

### 3.3 录同期声 / 静音分清

| 类型 | 是否同期声 | 备注 |
|---|---|---|
| **主动作演示** | ❌ **静音**（后期配旁白 + BGM）| `<video muted playsinline>` |
| **口播 / 出镜** | ✅ **同期声**（人声是主轴）| 后期只配 BGM |
| **数据 overlay** | ❌ 静音 | — |
| **B-roll** | ❌ 静音 | — |

> **默认拍"静音素材"**，因为后期可加旁白；拍"同期声"后再去噪很痛苦。

### 3.A A 类口播补充（v3 新增 · 2026-06-10）

> **A 类核心素材**。A 类是 3 类中**唯一必须出镜**的类型——人脸全程在场（口播态全屏 / 辅助素材态缩右下角圆头像）。**主口播视频同一份素材双用**。

#### 3.A.1 主口播视频拍摄规范

| 维度 | 取值 | 原因 |
|---|---|---|
| **数量** | 1 段贯穿全片（**≥ 60s 完整版**）| **双用**：口播态全屏 + 辅助素材态缩圆头像；不可剪断 |
| **机位** | **正面 / 1x 镜头 / 距离 1.2-1.5m** | 圆头像里"五官清晰"是底线，距离不能 > 2m |
| **取景** | **上半身 + 头部居中**（头顶留 ≥ 20% 空白）| 圆头像裁切时不能切到头顶/下巴 |
| **背景** | **干净单色 / 不杂乱**（与右下角 240px 圆头像对比清晰）| 圆头像里"边角不模糊" |
| **光线** | **顺光或 45° 侧光 / 主光在前方 45°** | 圆头像里"五官不阴阳脸" |
| **表情** | **口型清晰 / 有眼神交流 / 有手势** | 圆头像里 240px 直径下还能看出"在说话" |
| **声音** | ✅ **同期声**（人声是主轴，**轻后期降噪**）| 后期只配 BGM，不动人声 |
| **时长** | **≥ 60s**（一个完整版本，不剪断）| 圆头像需要**连续播放**才能"嘴在动"，**中途不能 pause** |
| **口型同步** | **每句旁白对应一段**——按 [copy.md 5 段文案](copy.md) 顺序录 | 保证圆头像"嘴在动"的进度 = 字幕进度 = **三同步** |

#### 3.A.2 录多版本（必走）

| 版本 | 用途 | 数量 |
|---|---|---|
| **主版本**（正面 / 1.2-1.5m / 上半身）| 1 段贯穿全片（必走）| 1 段 |
| **备用版本**（侧面 / 俯视 / 半身）| 后期剪辑替换 | 1-2 段 |

> **铁律**：**A 类视频必须先录主口播视频，再录辅助素材**——因为圆头像依赖主口播视频抽帧，**顺序反了素材就废了**。

#### 3.A.3 拍摄前 7 项检查（A 类专属补充）

在 [§2 7 项检查](#2-拍摄前-7-项检查) 基础上，A 类还必须额外检查：

```
A1. 主口播视频 ≥ 60s 完整版录好（不能中断）
A2. 口型清晰（嘴动幅度可见）—— 后期可识别"正在说话"
A3. 头顶留 ≥ 20% 空白 + 上下半身居中 —— 圆头像裁切时不切到关键部位
A4. 背景干净单色 —— 圆头像边角清晰
A5. 同期声清晰（同期声场景必查；A 类必是同期声）
A6. 主口播视频已就位（resources/audios/001_talking_head.mp4）—— 之后才能录辅助素材
A7. 表情 / 手势有 —— 圆头像里"有戏"
```

#### 3.A.4 拍摄后 4 项验收（A 类专属补充）

```
A1. 主口播视频时长 ≥ 60s（按 copy.md 5 段文案长度计算）
A2. 圆头像抽帧测试：随机抽 1 帧 → 240px 圆头像截图 → 五官清晰可见
A3. 同期声音量一致（不爆音 / 不突然变小）
A4. 主口播视频不暂停 / 不剪断 —— 圆头像播放必须连贯
```

#### 3.A.5 A 类拍摄顺序（与 B/C 不同）

```
Step 1：写 copy.md 5 段文案（按 v2 锁版：钩子 + 主体 3 段 + 收尾）
Step 2：录主口播视频（≥ 60s，按 5 段顺序录，**不剪断**）
Step 3：录辅助素材（屏幕录制 / 工具操作 / 录屏，5-10s/段）
Step 4：录 B-roll / 抽象动效（按需）
Step 5：验收 4 项（主口播视频 + 抽帧测试 + 同期声 + 连续性）
```

> **为什么 A 类顺序特别重要**：**圆头像依赖主口播视频的连续播放**。如果先录辅助素材再录主口播视频，辅助素材对应的口型/表情都对不上——后期改起来非常痛苦。

### 3.4 拍横平竖直 / 不歪

- iPhone 锁定自动旋转（控制中心 → 竖屏锁定）
- 主体居中（不偏左 / 偏右）
- 地平线水平（健身房镜子是天然参考）

## 4 · 拍摄后 4 项验收

```
[ ] 1. 文件能正常打开（QuickTime / VLC）
[ ] 2. 画面无明显抖动 / 失焦 / 曝光异常
[ ] 3. 声音无爆音 / 风声 / 杂音（如同期声）
[ ] 4. 时长 ≥ storyboard.md 要求（视频类 ≥ 5s）
```

> **验收失败 → 立即重拍**，不要想着"凑合用"。

### 4.1 验收详解

| # | 检查项 | 失败案例 | 修法 |
|---|---|---|---|
| 1 | 文件可打开 | 录制中断 / 损坏 | 重录 |
| 2 | 画面无抖动 | 手持 / 失焦 / 曝光 | 上三脚架 / 调光线 |
| 3 | 声音无杂音 | 风声 / 喷麦 / 杂音 | 换环境 / 调话筒距离 |
| 4 | 时长合规 | < 5s / 缺完整动作 | 重录完整动作 |

## 8 · 拍摄异常处理

### 8.1 常见异常 + 修法

| 异常 | 现象 | 修法 |
|---|---|---|
| **设备中途没电** | 录到一半关机 | 充电后重录（保持同一机位）|
| **存储满** | 录到一半无法保存 | 清存储后重录 |
| **光线突然变** | 视频明暗不一 | 重录 / 调灯 |
| **背景有人闯入** | 视频有路人 | 重录 / 裁切 |
| **动作做错** | 动作不规范 | 重录（不凑合）|
| **声音嘈杂** | 同期声听不清 | 换环境 / 后期去噪 |
| **时长不够** | 视频 < 5s | 重录完整动作 |

### 8.2 异常决策

| 异常 | 修法 | 阻塞时间 |
|---|---|---|
| **小问题**（1 处光线 / 1 个路人）| 重新拍 1 次 | +10 min |
| **大问题**（电量 / 存储）| 改天再拍 | +1 天 |
| **设备问题** | 换设备 | +1 天 |

## 9 · 5 维评分卡（拍摄质量）

> **每维 ≥ 3 分才能进 Phase 4（分镜实现）**，总分 ≥ 15/25。

| 维度 | 1 分（差）| 3 分（中）| 5 分（优）| 本稿得分 |
|---|---|---|---|---|
| **规格完整** | 5 维漏 1 项 | 5 维全填 | 5 维全填 + 扩展项填 | — |
| **拍摄前 7 项** | < 5 项打勾 | 5-6 项打勾 | 7 项全打勾 | — |
| **多角度版本** | 1 个版本 | 2 个版本 | 3+ 个版本 | — |
| **验收 4 项** | < 3 项过 | 3 项过 | 4 项全过 | — |
| **素材可用性** | 重拍 ≥ 2 次 | 重拍 1 次 | 一次过 | — |

## 11 · 反模式

- ❌ 5 维规格漏填就开拍（后期发现机位错 / 时长短）
- ❌ 手持拍主动作（明显抖动 → 视频不可用）
- ❌ 拍横屏（项目默认 1080×1920 竖屏，除非用户说"做横屏"）
- ❌ 同期声 / 静音不分（口播素材当动作素材用 = 后期无法配旁白）
- ❌ 拍单一角度（后期发现某个动作需要特写，没有素材）
- ❌ 拍完不验收直接进 scene.js 实现（发现文件坏 / 时长不够 = 返工）
- ❌ 在健身房 / 公共场地露出真实位置 / 同事（隐私泄露）
- ❌ 拍摄时穿浅色 / 印花服装（与 7fit 调性冲突，且后期难去色）
- ❌ **不拍多版本**（后期选择少）
- ❌ **不跑异常处理**（小问题变大问题）
- ❌ **跳过 5 维评分卡直接进实现**
