# 视频类型判定（video-types.md）

> **开工第一步**：用户说"做 X 视频"时，**先判断属于 3 类中的哪一类**，得到用户确认后再继续。
>
> 不判断类型 = 写不出合格的分镜 + BGM + 素材清单。
>
> **类型哲学**：**先判定类型，再写脚本**——3 类视频的"主体/目的/素材/BGM/CTA"差异极大（详 §7 对照表）。类型定错 = 整个流水线从根上错。3 要素：**判定必问**（不替用户决定）+ **混合类型有规则**（主体+辅助分两节）+ **禁止反查链**（A 类人设不引导到 C 类产品）。

---

## 1 · 3 类视频速查

| 类型 | 主体内容 | 辅助内容 | BGM 配对 | 占比示例 | 典型场景 |
|---|---|---|---|---|---|
| **A · 个人人设（OPC）** | **人脸 / 口播（用户自拍，必出镜）** + 左右分栏 | 视频/图片/代码/数据卡（**有辅助内容时视频缩到左侧 40%，辅助内容右侧 50%**）| **Quiet Think**（ambient, 80 BPM）| vlogs / 日常 / 工具分享 | "练了 3 年我才明白的道理" / "vibe coding 搭小程序" |
| **B · 健身知识** | 动作演示视频（用户自拍多角度）| 关键参数 overlay（动作名/重量/次数/RPE）| **Power Build**（tech house, 105 BPM）| 教学 / 科普 | "翼状肩 3 步自测 + 4 个动作" |
| **C · 七练介绍** | 弹出内容卡片（截图/操作视频/代码块/git log）| 弱化处理的健身视频 | **Cyber Pulse**（synthwave, 100 BPM）| 产品介绍 / 功能演示 | "我用 7 练小程序记了 100 次训练" |
| **D · 特效演示**（v3.5 新增）| 演示 layout / 动效 / 转场，**不承载内容** | （按 layout 需求）| 无强制（视效果定）| **布局状态机 / 动效库 / 内部参考** | 口播特效演示（`talking-head-effect-demo`）|

> **⚠️ A 类时长不限制**（用户 2026-06-15 决策 · v3.4）：A 类硬约束 = **≥ 90s 全文**（不是 60-90s 也不是 = 105.5s）。A 类是"长镜头 + 录屏"为主，时长由内容决定，**无强制上限**。详见 [§6 对照表"典型时长"行](#6--3-类视频的全套差异对照表核心新增)。
>
> **v3.3 → v3.4 变更**（2026-06-15）：A 类下限从 60s 提升到 **90s**——理由是 A 类需要更完整的故事弧（身份锚定→痛点→转折→解法→收益→升华），60s 装不下。

> **默认搭配**：A 视频类型 → C BGM 类型（Quiet Think）/ B 视频类型 → B BGM 类型（Power Build）/ C 视频类型 → A BGM 类型（Cyber Pulse）。详见 [bgm.md §默认搭配](bgm.md#21-默认搭配视频类型--bgm-类型)。

## 2 · A 类视频布局规范（必须遵循）

> **A 类（口播）视频的布局设计**，全部参考：
> [`remotion/src/layout-state-machine/index.ts`](remotion/src/layout-state-machine/index.ts)
>
> 该文件包含：10 种布局的可视化 ASCII 图示、精确坐标、所有设计约束。
> **设计 A 类脚本时，必须先读该文件，选择对应布局，再输出 shotSequence。**

### 2.1 布局选用速查

| 场景 | 推荐布局 | 特点 |
|---|---|---|
| 纯口播，无辅助 | `fullscreen` | 口播全屏 |
| 口播 + 文字/图片素材 | `left_text_right_talking` | 口播右侧 480px，辅助内容左侧 1340px |
| 口播 + 右下角色素材 | `pip_bottom_right` | 口播右下角小窗，素材全屏 |
| 口播 + 左下角色素材 | `pip_bottom_left` | 口播左下角小窗，素材全屏 |
| 多数据并列展示（≥4个素材） | `grid_2x2` | 口播左上格，4格辅助素材；素材不足时禁用，改用 `bottom_right_talking` |
| 口播 + 各占一半 | `left_text_right_talking_50pct` | 口播右半，辅助内容左半 |
| 口播为背景，内容聚焦 | `centered_fullscreen_bg` | 口播全屏铺底（半透明），内容居中叠加 |
| 口播居中，素材左右对称 | `center_dual_aux` | 口播居中(864²)，辅助内容左右各 400px |
| 口播居中，多视频环绕 | `orbiting_center` | 口播居中圆形，4 个视频沿轨道旋转 |

### 2.2 布局选用规则

- **每个 shot 只能选一种布局**，不能混用
- **口播全程不消失**（口播是身份锚点，任何时候都可见）
- **辅助内容 > 1 种时**，选 `center_dual_aux` 或 `orbiting_center`
- **isCircle=true** 时强制正圆渲染，可用于突出关键 shot

### 2.3 ShotEntry 格式

```ts
interface ShotEntry {
  shotId: string;
  layoutId: string;      // 选自上面 10 种之一
  transitionType: TransitionEasing;
  startFrame: number;
  endFrame: number;
  isCircle?: boolean;   // 可选，强制圆形口播
}
```

## 4 · 判定流程（必须问用户）

> **铁律**：**必须问用户**，不能凭"看起来像 X 类"开干。

```
用户：「做 X 视频」
  ↓
询问（4 个问题）：
  1. 主体是什么？
     - 人脸 / 口播 → A
     - 训练动作演示 → B
     - 产品功能 / 界面 → C
  2. 目的场景？
     - 涨粉 / 拉新 → A
     - 教学 / 科普 → B
     - 引流到小程序 → C
  3. 时长？
     - ≤ 30s → 单条短视频
     - 30-90s → 主推视频（B/C 类）
     - > 90s → 拆条发布（B/C 类）
     - **A 类：≥ 90s，无上限**（由口播内容决定时长）
  4. 关键素材是否就位？
     - A 必须有口播视频（用户自拍）
     - B 必须有训练动作视频（用户自拍，不信 mmx）
     - C 必须有弹出内容（截图/代码/git log）+ 背景健身视频
```

**确认后才能进入 Phase 1（写文案）。**

### 2.1 判定输出模板

> **确认后写进 `copy.md` 头部**（已锁版字段）：

```markdown
> 视频类型：A / B / C / D（D = 特效演示，v3.5 新增）
> 目的：涨粉 / 教学 / 转化 / 演示
> 时长：30-90s（B/C 类）/ ≥ 90s 无上限（A 类）/ 无约束（D 类）
> 关键素材：<列出就位情况>
> BGM 选型：<A/B/C/D 类>
```

> ⚠️ **A 类时长无上限**（≥ 90s 下限，无强制上限）——A 类以口播内容决定时长，不应以 90s 拆条。

## 5 · 画面占比硬约束

| 类型 | 主体占比 | 画面分配 |
|---|---|---|
| A · 个人人设 | 60-70%（口播段）/ 40%（辅助内容段）| 详见 §3.2 A 类双态布局（全屏态 + 左右分栏态）|
| B · 健身知识 | 动作 70%+ | 主体动作占主屏，辅助 = 关键参数 overlay（动作名/重量/次数/RPE）|
| C · 七练介绍 | 卡片 60% | 弹出卡片占主屏，背景 = 全画面弱化处理的健身视频 |

### 5.1 画面占比决策表

| 主体占比 | 辅助占比 | 适用类型 |
|---|---|---|
| 60-70% | 30-40% | **A 类口播段**（人脸为主，数据/动作为辅）|
| 40%（左侧视频）| 50%（右侧辅助）| **A 类辅助内容段**（视频缩到左侧 768×432，辅助内容右侧占 50%，5% 留白）|
| 70%+ | ≤ 30% | **B 类**（动作演示为绝对主体）|
| 60% | 40% | **C 类**（卡片+背景视频平衡）|

### 3.2 A 类双态布局（v4 锁版 · 2026-06-12）

> **A 类核心规则（v4 重写）**：**人物全程出镜**——全屏态时人脸占满屏，切换到辅助内容（视频/图片/代码/数据/录屏）时，**视频缩到左侧，辅助内容在右侧**，不再使用圆形 PIP。
> 核心目的不变：保持"个人人设"的身份认同——观众任何时候都能看见"谁在说"。

#### 3.2.1 两种状态

| 状态 | 视频位置 | 辅助内容位置 | 触发条件 | 比例 |
|---|---|---|---|---|
| **全屏态**（Full Screen）| 视频 x=0 y=0 w=1920 h=1080 铺满画布 | 仅字幕/标题栏 overlay | 纯口播 / 过渡 shot（无辅助素材）| 100% 视频 |
| **左右分栏态**（Split）| 视频 x=96 y=324 w=768 h=432（左侧 40%，5% 留白）| 辅助内容 x=960~1824 w=864（右侧 50%，含 5%右侧留白）| 旁白提到视频/图片/代码/数据/录屏/工具界面 | 40% 视频 + 50% 辅助 |

#### 3.2.2 左右分栏硬约束（v4 锁版）

| 维度 | 取值 | 备注 |
|---|---|---|
| **Split 视频位置** | `x=96, y=324`（左侧 5% 留白 = 1920×0.05）| 垂直居中：`y = (1080-432)/2` |
| **Split 视频尺寸** | `768×432`（16:9，画布宽度 40%）| 等比例缩放，保持 16:9 |
| **Split 辅助内容区** | `x=960, w=864`（右侧 50%）| 右侧留白 5%（`1824 = 1920×0.95`）|
| **全屏视频尺寸** | `1920×1080` 铺满 | 无圆角，无缩进 |
| **非横屏素材处理** | 画面居中 + 背景渐变/毛玻璃 | 见 §3.2.5 |
| **切换动效** | 0.5s `power2.inOut` 缩小/放大 + 位移 | 视频 w/h/x/y 同时变化，无 borderRadius |

#### 3.2.3 非横屏素材处理规则（v4 新增）

> **画布始终 1920×1080 横屏**。当用户提供的口播视频非 16:9 横屏时，按以下规则处理：

| 场景 | 视频处理 | 背景处理 | 布局 |
|---|---|---|---|
| **无辅助内容（全屏态）** | 画面居中，等比例缩放至高度填满 1080px | 画布铺满，背景用视频的放大 + 高斯模糊版本（`blur(40px)` + 暗色渐变 overlay）| 居中 |
| **有辅助内容（Split 态）** | 画面偏左，等比例缩放至 w=768（保持比例）| 辅助内容在右侧显示 | 左视频 + 右辅助 |

> **技术实现**：非横屏时，`.talking-head bg` 放一个额外的 `<div class="bg-blur">` 用同一视频源 + `filter: blur(40px) brightness(0.4)` + `transform: scale(1.2)` 铺满画布。
> 横屏素材（16:9）时无需此处理。

#### 3.2.4 辅助内容排列规则

> **辅助内容在 Split 态的右侧区域（x=960~1824），不是全屏展示。**

| 辅助内容类型 | Split 态右侧布局 | 典型场景 |
|---|---|---|
| **信息图/数据卡** | 右侧居中，垂直排列 | 5 大功能网格、工具栈对比 |
| **代码/录屏** | 右侧全高显示，可带半透明蒙版 | IDE 录屏、终端截图 |
| **过渡标题** | 全屏顶部 overlay（仅过渡 shot 使用）| 段间停顿 "→ 四步流程" |
| **CTA 卡片** | 右侧居中，大号文字 + 边框 | 评论区引导、关注引导 |

#### 3.2.5 切换动效

| 切换 | 时长 | ease | 实现 |
|---|---|---|---|
| 全屏 → Split | 0.5s | `power2.inOut` | 视频 `width: 1920→768, height: 1080→432, x: 0→96, y: 0→324` |
| Split → 全屏 | 0.5s | `power2.inOut` | 反向 |

> 详见 [animation.md §12 A 类双态切换动效](animation.md#12--a-类双态切换动效)

#### 3.2.6 四条布局硬约束（v4.1 新增 · 2026-06-13）

> 用户硬约束（2026-06-13），锁入规范。

| # | 约束 | 实现 |
|---|---|---|
| **a** | 口播视频**统一显示正方形**（无论提供横屏/竖屏） | 视频容器固定 `width: N, height: N`（Full Screen `1080×1080` / Split `574×574`）+ `video { object-fit: cover }` |
| **b** | Split 态视频占比 **< 30%**（1920×0.3 = 576px 上限） | Split 态视频宽度 ≤ 574px（`574/1920 = 29.9% < 30%`），当前 560px 符合规范 |
| **c** | A 类视频**每个场景必须有独立背景图** | 每场景一个 `<Img>` 组件，`Sequence` 入场时切换 `opacity`（当前场景显示，其余隐藏）；**禁止无背景图空场景** |
| **d** | Split 态**视频列无背景色/装饰效果** | Split 态时 `.talking-head` 的 `box-shadow` / `mask-image` 羽化效果关闭（`box-shadow: none; mask-image: none`），视频列直接透明叠在画布背景上 |

> **技术实现 c（per-scene 背景切换）**：
> ```tsx
> // 每个 Shot 有独立背景 Img，Sequence 入场时淡入
> <AbsoluteFill style={{ opacity: currentScene === 0 ? 1 : 0 }}>
>   <Img src={staticFile("bg_scene_0.png")} style={{ width: "100%", height: "100%" }} />
> </AbsoluteFill>
> <AbsoluteFill style={{ opacity: currentScene === 1 ? 1 : 0 }}>
>   <Img src={staticFile("bg_scene_1.png")} style={{ width: "100%", height: "100%" }} />
> </AbsoluteFill>
> // 用 frame 或 scene index 控制 opacity
> const bgOpacity = interpolate(sceneFrame, [0, 12], [0, 1], { extrapolateLeft: "clamp", ... });
> ```

### 3.3.3 布局技术实现（A类）

#### 3.3.3.1 全屏态（Full Screen）
```html
<!-- 全屏态：视频铺满画布 -->
<div class="talking-head full-screen" data-state="full-screen">
  <video class="talking-head__video" muted playsinline>
    <source src="<path>.mp4" type="video/mp4">
  </video>
</div>
```

```css
.talking-head.full-screen {
  position: absolute;
  top: 0; left: 0;
  width: 1920px; height: 1080px;
  border-radius: 0;
  overflow: hidden;
  /* 羽化描边（全屏态专用）*/
  box-shadow: inset 0 0 30px 8px rgba(10,10,10,0.5),
              0 0 80px 40px rgba(10,10,10,0.4);
  mask-image: radial-gradient(circle 80% 80% at 50% 50%, black 10%, transparent 100%);
}
```

```html
<!-- 左右分栏态：视频左 <30% + 辅助内容右 >70% -->

    <video class="talking-head__video" muted playsinline>
      <source src="<path>.mp4" type="video/mp4">
    </video>

  <div class="visual-support split-right" data-position="right">
    <div class="support-frame">
      <!-- 辅助内容：代码/数据/录屏 -->
    </div>
  </div>

```

```tsx
// 动作视频中央 + 四角参数卡随节拍脉冲
// ❌ 禁用 requestAnimationFrame（Remotion 用 useCurrentFrame + interpolate）
// ✅ Remotion 音频可视化：用 useAudioData from @remotion/media-utils
import { useAudioData, interpolate } from "remotion";

const BeatCard: React.FC = () => {
  const audioData = useAudioData(audioData);
  const frame = useCurrentFrame();

  // 用音频数据的 amplitude 驱动缩放
  const bass = audioData?.slice(0, 4).reduce((a, b) => a + b, 0) / 4 ?? 0;
  const scale = interpolate(frame, [0, 1], [1, 1 + bass / 500], {
    extrapolateLeft: "clamp",
    extrapolateRight: "clamp",
  });

  return (
    <div style={{ transform: `scale(${scale})` }}>
      {/* beat card content */}
    </div>
  );
};
```

## 14 · 反模式

- ❌ 没问用户就凭"看起来像 X 类"开干
- ❌ A 类视频用 BGM 重低音（健身房嗨曲），破坏人声清晰度
- ❌ B 类视频用 mmx 生成的假人健身演示（动作失真，用户一看就出戏）
- ❌ C 类视频没有内容卡片，只有产品截图（"这不是视频，是 PPT"）
- ❌ 混合类型不分主体 / 辅助，素材清单一团乱
- ❌ 切换类型时没有 ≥ 0.3s 段间停顿（硬切观众反应不过来）
- ❌ **类型判定后不写进 copy.md 头部**（下游没依据）
- ❌ **A 类人设引导到 C 类产品**（副号内容不允许带产品，详 §7.1）
- ❌ **混合类型没问用户主体就开干**
- ❌ **跳过 5 维评分卡直接写文案**

### 12.1 A 类专属反模式（v4 更新 · 2026-06-12）

- ❌ **A 类视频没有真人画面**（纯人声配风景/纯文字画面/纯产品截图）——**违反 §4 关键素材硬约束**
- ❌ **辅助内容态时人脸消失**（视频切到屏幕录制/图片后，没人脸了）——必须保留左侧视频
- ❌ **辅助内容全屏替代视频**（辅助素材铺满画布，视频消失）——A 类主体永远是人，视频必须在左侧可见
- ❌ **视频缩小使用圆形 PIP**（右下角圆头像，v3 旧布局）——v4 改为左右分栏，矩形视频，不用圆角
- ❌ **左右分栏时视频尺寸或位置不符合规范**（不是 768×432 或 x ≠ 96）——必须严格遵守 Split 态硬约束
- ❌ **双态切换无动画或 < 0.3s**（瞬间切换，生硬）——必须 0.5s `power2.inOut`
- ❌ **非横屏素材未处理背景**（竖屏口播视频直接居中，两侧黑边）——必须加渐变/毛玻璃背景
- ❌ **辅助内容在 Split 态超出右侧区域**（x < 960 或 w > 864）——必须遵守 5% 留白规则
- ❌ **口播视频未裁切为正方形**（竖屏素材直接居中显示，两侧留黑边）——必须用 `object-fit: cover` + 正方形容器
- ❌ **Split 态视频宽度 ≥ 576px**（占比 ≥ 30%）——必须 < 576px（当前锁版 ≤ 574px = 29.9%）
- ❌ **全片使用单一背景图**（A 类应有 per-scene 场景化背景）——必须为**每场景**生成独立 `bg_scene_<N>.png`；**禁止无背景图空场景**
- ❌ **Split 态视频列保留羽化描边/box-shadow 效果**（视频列应该无装饰，直接透明叠在画布背景上）——Split 态时关闭所有 feathering 效果

## 附录 C · 竖屏视频专项（2026-06-13 新增）

> 来源：短视频行业调研 2025-2026

### C.1 竖屏 vs 横屏选择

| 场景 | 推荐方向 | 原因 |
|---|---|---|
| 健身教学/跟练 | **竖屏 9:16** | 手机拍摄习惯 / 沉浸感 |
| 口播/人设 | 横屏 16:9 或竖屏 | 看平台和受众 |
| 产品展示 | **竖屏 9:16** | 短视频平台主流 |
| 横向对比/多器械 | 横屏或分屏 | 信息密度高 |

### C.2 竖屏构图原则

| 原则 | 描述 |
|---|---|
| **主体居中或偏上** | 手机视线习惯上部信息 |
| **安全区放大** | 竖屏裁剪风险高，上下左右各留 5% |
| **留白适当** | 太满显得拥挤 |
| **关键信息放中上部** | 底部容易被平台 UI 遮挡 |

### C.3 竖屏分屏布局

| 布局 | 比例 | 适用 |
|---|---|---|
| **上下分栏** | 50/50 | 动作示范 + 参数 |
| **左右分栏** | 40/60 | 人像 + 文字 |
| **画中画** | 80/20 | 主视角 + 细节 |
| **三分屏** | 30/40/30 | 多动作对比 |

### C.4 竖屏 B 类（健身）专项

| 技巧 | 描述 |
|---|---|
| **动作占 70%+** | 竖屏空间宝贵，动作是主体 |
| **参数悬浮右下/左下** | 不挡主体，清晰可读 |
| **底部留白 15%** | 防平台 UI 遮挡 |
| **竖屏手势示范** | 俯拍角度更易展示动作 |

### C.5 横竖屏转换处理

| 源素材 | 目标布局 | 处理方式 |
|---|---|---|
| 横屏口播 | 竖屏分栏 | 裁剪中部，左右可加辅助内容 |
| 横屏动作 | 竖屏全屏 | 智能裁剪，突出主体 |
| 竖屏口播 | 横屏 | 上下加装饰背景 |
